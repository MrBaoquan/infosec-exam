# 工业控制系统安全

## 1. 为什么工控安全不同于普通 IT 安全

某水厂的监控电脑感染勒索软件。普通办公电脑可以关机重装，但水泵控制系统突然停机会影响供水，错误指令还可能损坏设备或危及人员。

> 🎯 **形象比喻**：IT 系统主要处理“信息”，工业控制系统直接操纵“机器”。IT 故障可能丢文件，OT 故障可能停产、损坏设备或引发安全事故。

OT = Operational Technology（运营技术）。工控安全首先保障人员、设备和生产过程安全，其次才是传统的信息机密性目标。

## 2. 常见工控组件

| 组件 | 全称 | 作用 |
|------|------|------|
| **PLC** | Programmable Logic Controller | 按控制逻辑直接控制设备 |
| **DCS** | Distributed Control System | 对连续生产过程进行分布式控制 |
| **SCADA** | Supervisory Control and Data Acquisition | 远程监控、采集数据和下发控制指令 |
| **HMI** | Human-Machine Interface | 操作员查看状态和操作设备的界面 |
| **Historian** | Process Historian | 保存生产过程和设备历史数据 |
| **SIS** | Safety Instrumented System | 在危险状态下执行安全联锁和保护 |

简化的数据流为：

```text
现场传感器与执行器 ↔ PLC/DCS ↔ HMI/SCADA ↔ 生产管理系统 ↔ 企业管理网
```

越靠近现场设备，实时性和可用性要求越高；越靠近企业管理网，与互联网、邮件和办公终端接触越多，攻击入口也越多。

## 3. IT 与 OT 的安全差异

| 对比 | IT 信息系统 | OT 工业控制系统 |
|------|-------------|-----------------|
| 首要目标 | 数据机密性、完整性、可用性 | 人员与设备安全、过程完整性、可用性 |
| 生命周期 | 通常较短，便于更新 | 设备可运行十年以上，升级窗口少 |
| 实时性 | 一般可容忍短暂延迟 | 控制指令常有严格实时要求 |
| 补丁方式 | 可定期重启更新 | 需评估生产影响，在停机窗口验证 |
| 安全测试 | 可在测试环境扫描和渗透 | 主动扫描可能影响脆弱设备，需谨慎 |

> ⭐ 高频原则：工控系统不能照搬办公网的补丁和扫描策略。所有变更先验证兼容性，再在批准的维护窗口实施，并准备回退方案。

## 4. 典型威胁与攻击路径

### 企业网横向进入生产网

攻击者通过钓鱼邮件控制办公终端，再利用企业网与生产网之间过宽的访问权限进入控制区。

### 远程运维入口失控

供应商使用长期有效的共享账号或将远程桌面直接暴露在互联网，攻击者取得凭据后可直接操作 HMI 或工程师站。

### 移动介质和工程师站

U 盘、维护笔记本和工程软件可能把恶意代码带入相对封闭的生产网络。

### 不安全工业协议

部分传统协议最初假设网络环境可信，缺少认证、加密和完整性保护。攻击者进入网络后可能伪造读取或写入指令。

### 供应链与配置篡改

被篡改的固件、工程文件、控制逻辑或更新包可能绕过边界设备，直接影响控制过程。

## 5. 分区分域与工业 DMZ

不能简单地把企业网和生产网接在同一交换网络。应依据业务功能和风险划分安全区域，并只开放必要通信。

```text
互联网
  ↓
企业管理网
  ↓  防火墙
工业 DMZ：跳板机、补丁中转、日志与数据交换
  ↓  工业防火墙
生产监控区：SCADA、HMI、Historian
  ↓
现场控制区：PLC、DCS、传感器、执行器
```

工业 DMZ = Demilitarized Zone（隔离区），用于中转跨域访问和数据，避免企业网直接连接 PLC 或 HMI。

### 最小通信原则

- 默认拒绝跨区访问，只允许明确的源、目的、协议和方向；
- 企业网不能直接访问现场控制设备；
- 生产数据向上汇聚时优先使用单向传输或受控代理；
- 远程运维必须经过 VPN、MFA 和跳板机，并全程审计；
- 安全联锁系统与基本过程控制系统按风险独立保护。

## 6. 工控纵深防御

| 层面 | 主要措施 |
|------|----------|
| **资产与风险** | 建立资产、固件、协议和连接关系清单，识别关键控制回路 |
| **网络边界** | 分区分域、工业 DMZ、工业防火墙、单向隔离 |
| **身份访问** | 独立账号、最小权限、MFA、跳板机和操作审计 |
| **主机设备** | 应用白名单、关闭不必要服务、接口管控、安全基线 |
| **监测响应** | 被动流量监测、异常指令检测、集中日志和告警 |
| **维护恢复** | 受控补丁、配置与控制逻辑备份、离线恢复介质、演练 |

### 为什么常用白名单

HMI 和工程师站上的合法软件相对固定。应用白名单只允许批准程序运行，比单纯依赖病毒特征更适合稳定、变更少的工控主机。

### 为什么优先被动监测

部分旧设备和协议栈承受能力有限，主动扫描可能导致异常。被动监测通过镜像流量识别资产、协议和异常行为，对生产过程影响更小。

## 7. 工控事件响应

发现异常控制指令时，不能未经评估直接关闭所有设备。应由安全、生产、设备和安全管理人员共同决策：

1. 确认受影响区域、控制回路和安全后果；
2. 优先维持人员与设备安全，必要时切换到安全状态或人工控制；
3. 按区域隔离受影响网络，避免盲目全网断电；
4. 保全日志、工程文件、内存和网络流量等证据；
5. 清除恶意代码并从可信备份恢复配置和控制逻辑；
6. 验证工艺参数和安全联锁后，分阶段恢复生产；
7. 复盘攻击路径，收紧跨区策略和远程运维权限。

## 8. 易错辨析

### 易错1：隔离是否等于完全断网

- 工控隔离强调受控的数据交换和访问路径，不一定是永久物理断开
- 跨区通信应通过工业 DMZ、代理、网闸或单向传输等机制

### 易错2：工控系统是否应立即安装所有补丁

- 安全补丁重要，但必须先验证兼容性和生产影响
- 在维护窗口分批实施，并准备备份和回退方案

### 易错3：防火墙是否足够

- 防火墙只解决部分边界访问问题
- 仍需资产管理、身份认证、白名单、监测、备份和响应演练

### 易错4：工控安全的首要目标

- 不能只按 CIA 三性机械排序
- 人员安全、设备安全、过程完整性和持续生产通常优先

## 9. 真题示例及解析

### 真题1（上午·单选）

> 工业控制系统中，为减少企业办公网被攻陷后直接访问PLC的风险，最合适的架构措施是？
> A. 所有设备使用同一网段  B. 在企业网和生产网之间设置工业DMZ并执行最小通信策略  C. 仅隐藏设备IP  D. 允许供应商直接远程桌面访问

**答案：B**

**解析**：工业DMZ用于承载跳板、数据交换和日志等跨域服务，结合防火墙最小通信策略，可阻断企业网到现场控制设备的直接路径。同网段会扩大横向移动风险；隐藏IP不能代替访问控制；直接暴露远程桌面风险极高。

---

### 真题2（下午·简答）

> 某工厂允许供应商通过共享账号从互联网直接远程访问工程师站，企业网与生产网之间只部署一台允许双向访问的防火墙。请指出主要风险并提出五项整改措施。

**参考答案与评分点：**

1. 共享账号无法追责，互联网直连和过宽双向策略可能使攻击者直接进入生产区。（2分）
2. 建立企业网、工业DMZ、生产监控区和现场控制区的分区分域架构。（2分）
3. 远程运维经过 VPN、MFA 和跳板机，使用个人账号、临时授权和全程审计。（2分）
4. 跨区策略默认拒绝，只开放必要源、目的、协议和方向，禁止企业网直达 PLC。（2分）
5. 对工程师站实施应用白名单、移动介质管控和安全基线。（1分）
6. 部署被动监测，备份控制逻辑和配置，并定期开展恢复演练。（1分）

## 10. 考点速记

- OT 直接控制物理过程，人员、设备、过程完整性和可用性优先
- 典型组件：PLC、DCS、SCADA、HMI、Historian、SIS
- 企业网与生产网之间设置工业 DMZ，跨区访问默认拒绝、按需放行
- 远程运维使用 VPN、MFA、跳板机、个人账号和全程审计
- 工控补丁先验证再在维护窗口实施；监测优先采用被动方式
- 应用白名单适合软件稳定的工控主机；配置和控制逻辑必须有可信备份
