# 云安全与物联网安全

## 云计算服务模型

| 模型 | 提供内容 | 用户管理 | 典型 |
|------|----------|----------|------|
| **IaaS** | 基础设施（虚拟机/网络/存储） | OS+中间件+应用 | AWS EC2 |
| **PaaS** | 平台（OS+运行环境） | 应用+数据 | App Engine |
| **SaaS** | 软件（完整应用） | 仅使用 | Office365 |

## 责任共担模型（高频）

| 模型 | 云厂商负责 | 用户负责 |
|------|-----------|----------|
| **IaaS** | 硬件/虚拟化/网络 | **OS+应用+数据** |
| **PaaS** | 硬件/虚拟化/OS/平台 | **应用+数据** |
| **SaaS** | 硬件/虚拟化/OS/应用 | **数据+访问配置** |

> 🔑 核心规律：**越往上(SaaS)云厂商负责越多；数据安全始终用户负责**。

## CSA 云安全顶威胁

1. 数据泄露
2. 身份与访问管理失效
3. 不安全的接口/API
4. 系统漏洞
5. 账号劫持
6. 恶意内部人员
7. APT
8. 数据丢失
9. 尽职调查不足
10. 滥用/恶意使用云服务

## 云安全措施

- **身份与访问管理 IAM**：最小权限、MFA
- **数据加密**：传输加密(TLS)+存储加密(KMS)
- **网络隔离**：VPC、安全组
- **日志审计**：CloudTrail 等
- **配置基线**：CIS Benchmark

## 物联网安全威胁

| 层次 | 威胁 |
|------|------|
| 感知层 | 设备被控、伪造数据、物理破坏 |
| 网络层 | 中间人、DoS、窃听 |
| 应用层 | 数据泄露、隐私、认证弱 |

### IoT 特有安全挑战

- **设备资源受限**：难运行复杂加密
- **数量巨大**：管理难、批量漏洞
- **物理暴露**：易被接触篡改
- **固件老旧**：补丁难推送
- **默认弱口令**：大量设备未改默认密码（Mirai 僵尸网）

## IoT 安全防护

1. 设备认证与密钥管理
2. 通信加密
3. 固件签名与安全更新
4. 最小权限
5. 网络分段隔离
6. 安全网关

## 考点速记

- IaaS 用户管 OS+应用+数据；SaaS 用户仅管数据
- **数据安全始终用户负责**
- CSA 顶威胁：数据泄露排第一
- IoT 挑战：资源受限、数量大、物理暴露、默认弱口令
- Mirai 僵尸网利用**默认弱口令**感染 IoT 设备
