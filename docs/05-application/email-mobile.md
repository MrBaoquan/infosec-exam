# 邮件与移动安全

## 邮件安全威胁

| 威胁 | 说明 |
|------|------|
| 钓鱼邮件 | 伪造发件人诱导 |
| 垃圾邮件 | 批量骚扰 |
| 邮件劫持 | 中间人篡改 |
| 邮件泄露 | 明文传输被窃听 |
| 恶意附件 | 携带病毒/木马 |

## 邮件加密方案

| 方案 | 信任模型 | 特点 |
|------|----------|------|
| **PGP** | 信任网(Web of Trust) | 去中心化，用户互相签名 |
| **S/MIME** | 基于 PKI 证书 | 中心化 CA，企业常用 |

- 都提供：机密性（加密）+ 完整性 + 认证 + 不可否认（数字签名）
- PGP 用 Web of Trust（无 CA）；S/MIME 用 X.509 证书（有 CA）

## 邮件防伪造三件套

| 技术 | 作用 |
|------|------|
| **SPF** | 声明合法发件服务器 IP，防伪造源 |
| **DKIM** | 邮件数字签名，防篡改+防伪造 |
| **DMARC** | 综合 SPF+DKIM，告知接收方如何处理伪造 |

## 移动安全威胁

| 威胁 | 说明 |
|------|------|
| 恶意 App | 窃取隐私、扣费、广告 |
| 重打包 | 篡改合法 App 植入恶意代码 |
| 不安全通信 | 明文传输、弱加密 |
| 越狱/Root | 突破沙箱，提权 |
| 系统漏洞 | 提权漏洞利用 |

## Android 安全机制

| 机制 | 说明 |
|------|------|
| **沙箱** | 每个 App 独立 UID，互不干扰 |
| **签名** | App 需签名，防篡改 |
| **权限** | 运行时动态申请敏感权限 |
| **SELinux** | 强制访问控制 |
| **Verify Apps** | 扫描恶意应用 |

## iOS 安全机制

| 机制 | 说明 |
|------|------|
| **沙箱** | 比 Android 更严格 |
| **代码签名** | 必须 Apple 签名 |
| **App Store 审核** | 上架前审查 |
| **数据保护** | 硬件加密（Secure Enclave） |
| **越狱检测** | 部分应用拒绝越狱设备 |

## 移动 App 加固

- 代码混淆（防逆向）
- 反调试
- 完整性校验（防篡改）
- 通信加密（防窃听）
- 数据本地加密存储

## 5. 易错辨析

### 易错1：PGP vs S/MIME
- **PGP**：**信任网**(Web of Trust)，无CA，去中心化
- **S/MIME**：基于**PKI证书**(X.509)，有CA，企业常用
- 考题："哪个无CA用信任网"→PGP

### 易错2：SPF/DKIM/DMARC
- **SPF**：声明合法发件服务器IP（防伪造源）
- **DKIM**：邮件数字签名（防篡改+防伪造）
- **DMARC**：综合SPF+DKIM，告知如何处理伪造
- 三者都是防邮件伪造

### 易错3：Android vs iOS 沙箱
- Android：基于UID沙箱，权限运行时申请
- iOS：沙箱更严格+Apple签名+上架审核
- iOS比Android封闭但更安全

## 6. 真题示例及解析

### 真题1（上午·单选）

> PGP 邮件加密使用的信任模型是？
> A. PKI证书  B. 信任网(Web of Trust)  C. CA中心  D. KDC

**答案：B**

**解析**：PGP用信任网(无CA，用户互相签名)。S/MIME用PKI/X.509证书(有CA)。

---

### 真题2（上午·单选）

> Android 系统的沙箱机制基于？
> A. 文件权限  B. UID(每个App独立UID)  C. 虚拟机  D. 加密

**答案：B**

**解析**：Android每个App分配独立UID，沙箱隔离基于UID。iOS沙箱更严格。

## 7. 考点速记

- **PGP 用信任网（无CA）**；**S/MIME 用 PKI 证书（有CA）**
- 邮件防伪造：**SPF+DKIM+DMARC**
- Android 沙箱基于 UID；iOS 沙箱更严格 + Apple 签名
- Android 权限**运行时动态申请**
- App 加固：混淆+反调试+完整性校验
