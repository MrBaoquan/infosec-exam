# 数据库与中间件安全

## 数据库安全威胁

| 威胁 | 说明 |
|------|------|
| SQL 注入 | 见 Web 攻击章节 |
| 越权访问 | 权限配置不当 |
| 数据泄露 | 明文存储、备份泄露 |
| 拖库 | 数据库被整体窃取 |
| 提权 | 利用漏洞获取 DBA 权限 |

## 数据库安全措施

| 措施 | 说明 |
|------|------|
| **最小权限** | 应用账号只授权必要表/操作，禁用 DBA |
| **视图** | 用视图隔离列，限制可见数据 |
| **加密存储** | 敏感字段加密、**透明数据加密 TDE** |
| **审计日志** | 记录敏感操作 |
| **备份恢复** | 全量+增量+日志 |
| **网络隔离** | 数据库不暴露公网，仅内网访问 |
| **脱敏** | 测试环境用脱敏数据 |

## 数据库加密层次

| 层次 | 方式 | 特点 |
|------|------|------|
| 字段级 | 应用层加密特定字段 | 灵活但影响查询 |
| **透明加密 TDE** | 数据库引擎层加密 | **应用无感知**，对查询透明 |
| 存储级 | 磁盘加密 | 防物理窃取 |

## Web 服务器安全（Apache/Nginx）

| 措施 | 说明 |
|------|------|
| 隐藏版本号 | `ServerTokens Prod` |
| 最小模块 | 只启用必要模块 |
| HTTPS | 强制 TLS |
| 安全头 | HSTS、X-Frame-Options、X-Content-Type-Options |
| 目录权限 | Web 目录最小权限 |
| 日志 | 访问日志 + 错误日志 |
| WAF | 防常见 Web 攻击 |

## 中间件安全

- **反序列化**：Java/Python 反序列化漏洞（WebLogic、Fastjson）
- **弱口令**：管理后台默认口令
- **漏洞**：及时打补丁（如 Log4j2 漏洞）

## 5. 易错辨析

### 易错1：TDE 透明加密
- **TDE**(Transparent Data Encryption)：数据库引擎层加密
- 特点：**应用无感知**（对查询透明）
- 不是字段级加密（字段级需应用改代码）

### 易错2：数据库账号权限
- 应用账号应**最小权限**（禁用DBA）
- 只授权必要的表和操作
- 考题："数据库安全第一步"→最小权限

### 易错3：脱敏
- 测试环境**必须用脱敏数据**
- 不能直接用生产数据测试
- 脱敏方式：替换/屏蔽/哈希

## 6. 真题示例及解析

### 真题1（上午·单选）

> 数据库透明加密(TDE)的特点是？
> A. 需修改应用代码  B. 应用无感知，引擎层加密  C. 只加密日志  D. 降低安全性

**答案：B**

**解析**：TDE在数据库引擎层加密，**应用无感知**，对查询透明。字段级加密才需改应用代码。

---

### 真题2（上午·单选）

> 数据库应用账号的安全原则是？
> A. 授予DBA权限  B. 最小权限，只授权必要表  C. 使用root  D. 共享账号

**答案：B**

**解析**：数据库应用账号应最小权限，只授权必要的表和操作，禁用DBA/root。防SQL注入后提权。

## 7. 考点速记

- 数据库最小权限：应用账号禁用 DBA
- **TDE 透明加密**对应用无感知
- 数据库不暴露公网，仅内网访问
- Web 服务器：隐藏版本、安全头、WAF
- 反序列化漏洞：WebLogic/Fastjson/Log4j2 典型
