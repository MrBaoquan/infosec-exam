# Web 安全攻击详解

> 🎯 **形象比喻**：Web 应用像一家"对外开放的银行"——
>
> - **SQL 注入** = 业务员把客户填的表单直接念给保险柜密码（输入被当指令执行）
> - **XSS** = 客户在表单里塞了张小纸条，下一个客户看到就中招（恶意脚本注入页面）
> - **CSRF** = 骗子借你的工牌刷门禁（借已登录身份发请求，不偷 Cookie）
> - **文件上传** = 有人把炸弹伪装成快递投递进仓库（上传 WebShell）

## OWASP Top 10

> OWASP = Open Web Application Security Project（开放 Web 应用安全项目）
> 注入、失效认证、敏感数据泄露、XXE、失效访问控制、安全配置错误、XSS、不安全反序列化、使用含漏洞组件、日志监控不足。

## SQL 注入（下午大题高频）

- 原理：用户输入**拼入 SQL**，作为代码执行
- 危害：拖库、绕过登录、提权、执行系统命令
- 类型：布尔盲注、时间盲注、联合查询、堆叠查询

<SqlInjection />

**防护**：

1. **参数化查询/预编译**（根本措施）
2. 输入校验（白名单）
3. 最小权限（数据库账号低权限）
4. WAF
5. 错误信息屏蔽

## XSS 跨站脚本

| 类型       | 存储位置 | 触发           | 危害程度 |
| ---------- | -------- | -------------- | -------- |
| **存储型** | 数据库   | 他人访问即触发 | 高       |
| **反射型** | URL 参数 | 用户点击链接   | 中       |
| **DOM 型** | 前端 JS  | 前端渲染触发   | 中       |

- 危害：盗 Cookie、会话劫持、键盘记录、挂马、XSS 钓鱼

**防护**：

1. **输出编码**（转义 `<>&"'`）— 核心
2. **CSP**（内容安全策略）
3. **HttpOnly Cookie**（防 JS 读取 Cookie）
4. 输入校验

## CSRF 跨站请求伪造

- 原理：借助受害者**已登录身份**，诱导其发起请求
- 与 XSS 区别：CSRF **借身份**（不偷 Cookie），XSS **偷身份**

**防护**：

1. **Anti-CSRF Token**（主流）
2. Referer 校验
3. **SameSite Cookie** 属性
4. 二次确认（敏感操作）

## 文件上传漏洞

- 危害：上传 WebShell、木马，获取服务器控制
- 防护：
    1. **白名单**校验扩展名（非黑名单）
    2. 校验 MIME 类型
    3. 校验文件头（魔数）
    4. **重命名**（避免原名）
    5. 存储到**非 Web 目录**（不可执行）
    6. 限制大小

## 文件包含漏洞

- 原理：动态包含文件时，用户可控路径
- 类型：本地包含(LFI)、远程包含(RFI)
- 防护：关闭远程包含、白名单、固定路径

## SSRF 服务端请求伪造

- 原理：服务端发起请求时，用户可控目标 URL
- 危害：访问内网、探测端口、读取本地文件
- 防护：**白名单限制内网访问**、禁用危险协议(file://)

## 越权访问

| 类型         | 说明                                       |
| ------------ | ------------------------------------------ |
| **水平越权** | 同级别用户互相访问数据（A 看 B 的订单）    |
| **垂直越权** | 低权限访问高权限功能（普通用户进管理后台） |

- 防护：**每次操作都校验权限**（不能只靠前端隐藏菜单）

## 反序列化漏洞

- 原理：反序列化不可信数据时执行恶意代码
- 防护：不反序列化不可信数据、白名单类

## 5. 易错辨析

### 易错1：SQL注入 vs XSS vs CSRF 核心防护

| 攻击    | 根因           | 核心防护            |
| ------- | -------------- | ------------------- |
| SQL注入 | 输入拼入SQL    | **参数化查询**      |
| XSS     | 输入未编码输出 | **输出编码**        |
| CSRF    | 借身份发请求   | **Anti-CSRF Token** |

### 易错2：CSRF vs XSS 偷不偷Cookie

- **CSRF**：**借身份**（不偷Cookie，借已登录状态发请求）
- **XSS**：**偷身份**（偷Cookie/执行脚本）
- 考题："不偷Cookie但借身份"→CSRF

### 易错3：文件上传防护

- 用**白名单**非黑名单（黑名单易绕过）
- **重命名**（避免原名攻击）
- 存**非Web目录**（不可执行）

### 易错4：水平 vs 垂直越权

- **水平**：同级互访（A看B的订单）
- **垂直**：低权限访问高权限（提权）
- 防：**每次操作校验权限**

## 6. 真题示例及解析

### 真题1（上午·单选）

> 防御 SQL 注入最根本的措施是？
> A. 输入过滤 B. 参数化查询 C. 前端校验 D. 隐藏错误

**答案：B**

**解析**：参数化查询/预编译从根本上隔离代码与数据，输入只作数据不作SQL代码。输入过滤是辅助。

---

### 真题2（上午·单选）

> XSS 跨站脚本攻击的核心防护措施是？
> A. 参数化查询 B. 输出编码 C. CSRF Token D. HTTPS

**答案：B**

**解析**：XSS防输出编码（转义HTML特殊字符）+CSP+HttpOnly Cookie。参数化查SQL注入，Token防CSRF。

---

### 真题3（上午·单选）

> CSRF 与 XSS 的关键区别是？
> A. CSRF偷Cookie，XSS借身份 B. CSRF借身份，XSS偷身份 C. 两者相同 D. CSRF加密

**答案：B**

**解析**：CSRF借受害者已登录身份发请求（不偷Cookie）；XSS偷Cookie/执行脚本。防CSRF用Token，防XSS用输出编码。

---

### 真题4（上午·单选）

> 用户A能查看同级用户B的数据，这属于？
> A. 垂直越权 B. 水平越权 C. SQL注入 D. XSS

**答案：B**

**解析**：水平越权=同级别互访；垂直越权=低权限访问高权限（提权）。每次操作都应校验权限。

---

### 真题5（上午·单选）

> 文件上传漏洞最有效的防护是？
> A. 黑名单校验 B. 白名单+重命名+存非Web目录 C. 仅校验MIME D. 仅限制大小

**答案：B**

**解析**：白名单（非黑名单）+重命名+存不可执行目录+校验文件头。黑名单易绕过，仅MIME可伪造。

## 7. 考点速记

- SQL注入防：**参数化查询**；XSS防：**输出编码+CSP+HttpOnly**；CSRF防：**Token**
- XSS 三型：**存储型(最危险)/反射型/DOM型**
- CSRF 借身份（不偷），XSS 偷身份
- 文件上传防：**白名单+重命名+非Web目录**
- 越权：水平=同级互访，垂直=提权；每次操作校验权限
- SSRF 防：白名单限内网
