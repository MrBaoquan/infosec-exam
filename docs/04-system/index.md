# 四、系统安全

## 操作系统安全机制

- **身份认证**：口令、生物特征、双因素
- **访问控制**：见下表
- **最小权限原则**：仅授予完成任务所需最小权限
- **安全审计**：记录日志，可追溯

## 访问控制模型

| 模型 | 机制 | 典型 |
|------|------|------|
| **DAC** 自主访问控制 | 资源所有者决定 | Linux 文件权限 |
| **MAC** 强制访问控制 | 系统按密级强制 | BLP/Biba、军网 |
| **RBAC** 基于角色 | 用户→角色→权限 | 企业系统主流 |
| **ABAC** 基于属性 | 属性策略动态 | 云环境 |

## 恶意代码

| 类型 | 行为 |
|------|------|
| **病毒** | 需宿主，依附传播 |
| **蠕虫** | 独立运行，主动传播 |
| **木马** | 伪装正常程序，开后门 |
| **后门** | 绕过认证秘密访问 |
| **Rootkit** | 隐藏自身，提权驻留 |
| **勒索软件** | 加密文件勒索赎金 |
| **逻辑炸弹** | 满足条件触发 |
| **间谍软件** | 窃取信息 |

## 漏洞管理

- **CVE** 公共漏洞编号
- **CVSS** 评分（0~10）
- 生命周期：发现→分析→修复→验证→发布补丁
- **0day** 尚无补丁的漏洞

## Windows / Linux 安全

- Windows：UAC、组策略、BitLocker、EFS
- Linux：SELinux/AppArmor(MAC)、sudo、chmod、iptables

## 可信计算

- **TPM** 可信平台模块 — 硬件根可信
- **TCM** 国产可信密码模块
- 信任链：CRTM→BIOS→OS→应用，逐级度量

## 考点速记

- DAC 所有者决定(灵活不安全)；MAC 系统强制(严格)；RBAC 角色中介(企业常用)
- 病毒需宿主，蠕虫独立传播，木马伪装后门
- 0day = 无补丁漏洞；CVE 编号，CVSS 评分
- TPM 是可信计算硬件根
