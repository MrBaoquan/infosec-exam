# 安全运营、取证与标准补充

> 本篇补充安全设备技术指标、运维指标、电子取证、相关标准。标注 ⭐ 高频，📌 了解即可。

## 安全设备技术指标 ⭐

### 性能指标
| 指标 | 说明 |
|------|------|
| **吞吐量** | 每秒处理数据量（Gbps） |
| **并发连接数** | 同时维持的连接数 |
| **新建连接速率** | 每秒新建连接数（CPS） |
| **规则数/特征数** | 支持的检测规则数量 |
| **延迟** | 处理引入的延迟（ms） |

### 检测指标（IDS/IPS）
| 指标 | 说明 |
|------|------|
| **检出率** | 正确检出的攻击比例（越高越好） |
| **误报率**（False Positive） | 正常被误判为攻击（越低越好） |
| **漏报率**（False Negative） | 攻击被漏判为正常（越低越好） |

> 💡 误报率与漏报率互相制约：降低一个会升高另一个。

## 运维指标 📌

| 指标 | 全称 | 说明 |
|------|------|------|
| **MTBF** | Mean Time Between Failures | 平均无故障时间（越长越可靠） |
| **MTTR** | Mean Time To Repair | 平均修复时间（越短越好） |
| **SLA** | Service Level Agreement | 服务级别协议（如99.9%可用） |
| **SLO** | Service Level Objective | 服务级别目标 |

## 电子取证 ⭐

> 🎯 电子取证像"犯罪现场勘查"——但证据是数字的，易丢失、易篡改，需严格保全。

### 取证原则
1. **易失性优先**：内存→磁盘（关机前先抓内存）
2. **完整性保全**：哈希校验，证据链完整
3. **不修改原件**：对镜像副本分析，不动原始介质
4. **全程记录**：记录操作的每一步

### 证据来源
| 来源 | 说明 |
|------|------|
| 内存镜像 | 运行进程/网络连接/密钥（易失） |
| 磁盘镜像 | 文件/注册表/日志 |
| 网络抓包 | 流量记录（pcap） |
| 系统日志 | 事件记录 |

### 取证工具
| 工具 | 用途 |
|------|------|
| **Volatility** | 内存取证分析 |
| **Autopsy** | 磁盘取证（开源） |
| **EnCase** | 商业取证套件 |
| **FTK** | Forensic Toolkit |
| **grep/find** | 文本搜索 |

### 证据链（证据保管链）
- 从采集到呈堂的全过程记录
- 任何环节断裂→证据可能不被采信
- 每次交接记录：时间、人、操作

## 认证协议补充 ⭐

| 协议 | 说明 |
|------|------|
| **RADIUS** | 远程认证拨号用户服务，UDP 1812/1813 |
| **TACACS+** | 终端访问控制器，TCP 49，比RADIUS更细粒度 |
| **LDAP** | 轻量目录访问协议，目录服务标准 |
| **PAP** | 密码认证协议（明文传密码，不安全） |
| **CHAP** | 挑战握手认证（加密，安全） |

### RADIUS vs TACACS+
| 对比 | RADIUS | TACACS+ |
|------|--------|---------|
| 协议 | UDP | TCP |
| 认证/授权/审计 | 合并 | **分离** |
| 加密 | 仅密码 | **全部加密** |

## 网络地址转换（NAT）📌

- **NAT**：将内网私有IP转换为公网IP
- **PAT**（端口地址转换）：多内网IP共享一个公网IP+不同端口
- 安全意义：隐藏内网结构

## 分组密码补充 ⭐

### 初始化向量（IV）
- CBC/CFB/OFB/CTR模式需要IV
- IV作用：相同明文+相同密钥→不同密文（防模式分析）
- IV要求：**随机/不可预测**，无需保密但需唯一
- ECB模式**不需要IV**（也因此不安全）

## IKE 协商模式 📌

| 模式 | 说明 |
|------|------|
| **主模式** | 6步，身份保护，更安全 |
| **野蛮模式**（Aggressive） | 3步，快但暴露身份 |
| **快速模式** | IPSec SA协商（第二阶段） |

## 考点速记
- 性能指标：吞吐量(Gbps)/并发连接/新建连接(CPS)
- 检测指标：检出率(高好)/误报率(低好)/漏报率(低好)，互相制约
- MTBF越长越可靠；MTTR越短越好
- 取证：**易失性优先**(内存→磁盘)、哈希校验、证据链
- 取证工具：Volatility(内存)/Autopsy(磁盘)/EnCase(商业)
- RADIUS(UDP/合并) vs TACACS+(TCP/分离/全加密)
- PAP明文不安全；CHAP挑战握手安全
- IV需随机唯一，ECB不需要IV(不安全)
- ISO 22301=业务连续性；NIST SP 800-53=安全控制
- NAT隐藏内网；PAT多IP共享一端口
