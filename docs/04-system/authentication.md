# 认证技术原理与应用

> 🎯 **形象比喻**：认证像"证明你是你"——口令(你知道什么)、智能卡(你拥有什么)、指纹(你是什么)，多因素组合更安全。

## 认证三因素

| 因素 | 类型 | 代表 |
|------|------|------|
| **你知道什么** | 知识因素 | 口令、PIN、安全问题 |
| **你拥有什么** | 持有因素 | 智能卡、U盾、手机(短信) |
| **你是什么** | 生物因素 | 指纹、人脸、虹膜、声纹 |

> **双因素认证(2FA)** = 两个不同类因素组合（如 口令+短信），安全性远高于单口令。

## 认证类型

| 类型 | 说明 | 典型 |
|------|------|------|
| **单向认证** | 只验证一方 | 客户端验证服务器(HTTPS) |
| **双向认证** | 双方互验 | 网银客户端+服务器互验证书 |
| **第三方认证** | 引入可信第三方 | Kerberos(KDC)、PKI(CA) |

## 认证技术方法

### 1. 口令认证
- 最常用但最弱
- 存储：**哈希+盐**（防彩虹表）
- 慢哈希：bcrypt/scrypt/PBKDF2（增加破解成本）
- 风险：口令泄露、弱口令、中间人
- 增强：失败锁定、口令复杂度策略

### 2. 智能卡(IC卡)认证
- **双因素**：卡(持有)+PIN(知识)
- 卡内存储证书/密钥，不可复制
- 比口令更安全，但需硬件
- 代表：U盾、PKI智能卡

### 3. 生物特征认证
| 特征 | 特点 |
|------|------|
| **指纹** | 最成熟，应用广 |
| **人脸** | 便捷，但可能被照片欺骗 |
| **虹膜** | 准确率最高，设备贵 |
| **声纹** | 非接触，易受噪声影响 |

- 优点：不易遗忘/被盗
- 缺点：**不可更改**（泄露即永久）、误识率(FAR)/拒识率(FRR)
- 隐私：生物数据敏感，需加密存储

### 4. Kerberos 认证
- 基于**对称密钥** + KDC 第三方
- 用**票据(Ticket)** 认证，避免反复传密码
- 详见[Kerberos详解](../crypto/kerberos-x509)

### 5. PKI 认证
- 基于**非对称密钥** + CA 证书
- 公钥验证签名/加密
- 详见[PKI章节](../crypto/pki)

### 6. 单点登录(SSO)
> 🎯 **形象比喻**：SSO 像"一次签证通行多国"——登录一次，访问所有授权系统，不用每个系统都登录。

| 类型 | 说明 | 代表 |
|------|------|------|
| **企业SSO** | 域内统一认证 | Windows AD、Kerberos |
| **Web SSO** | 跨站点统一登录 | OAuth2、SAML、OpenID Connect |

**SSO 原理**：
1. 用户登录**身份提供者(IdP)** 一次
2. 各**服务提供者(SP)** 信任 IdP 的令牌
3. 用户访问 SP 时，SP 向 IdP 验证令牌，无需再次登录

**SSO 优点**：用户体验好、减少口令管理负担
**SSO 风险**：IdP 是单点，一旦被攻破所有系统沦陷 → 需加强 IdP 安全(多因素)

### 7. 网络接入认证协议

| 协议 | 工作方式 | 关键考点 |
|------|----------|----------|
| **PAP** | 直接发送用户名和口令 | 口令可被链路窃听，不安全 |
| **CHAP** | 服务端发送随机挑战，客户端返回基于共享秘密计算的响应 | 口令不直接在链路上传输，但不加密后续业务数据 |
| **RADIUS** | 集中处理网络接入的认证、授权和计费，常用 UDP 1812/1813 | 认证与授权处理较紧密，只保护口令字段 |
| **TACACS+** | 集中管理网络设备访问，使用 TCP 49 | 认证、授权、审计可分离，保护完整报文主体 |
| **LDAP** | 访问目录中的用户、组和组织信息 | 是目录访问协议，不等同于完整认证方案 |

> 💡 CHAP 的“挑战—响应”用于证明客户端知道共享秘密。它避免直接发送明文口令，但不能称为“加密通信”。

#### RADIUS vs TACACS+

| 对比 | RADIUS | TACACS+ |
|------|--------|---------|
| 传输 | UDP | TCP |
| 典型场景 | 用户接入网络 | 网络设备管理 |
| AAA处理 | 认证和授权通常结合 | 认证、授权、审计可分离 |
| 报文保护 | 主要保护口令字段 | 保护报文主体 |

## 认证技术指标

| 指标 | 说明 |
|------|------|
| **FAR** False Accept Rate | 误识率（认错人），越低越安全 |
| **FRR** False Reject Rate | 拒识率（认不出），越低体验越好 |
| **EER** Equal Error Rate | 等错误率(FAR=FRR)，综合评估 |

> FAR 与 FRR 互相制约：降低一个会升高另一个。

## 5. 易错辨析

### 易错1：认证三因素组合
- 双因素 = 两个**不同类**因素组合
- 口令+短信 = 知识+持有 ✅
- 口令+安全问题 = 知识+知识 ❌（同类，不算双因素）

### 易错2：FAR vs FRR
- **FAR**(误识率)：认错人，越低越**安全**
- **FRR**(拒识率)：认不出，越低**体验**越好
- 两者互相制约，降低一个会升高另一个

### 易错3：SSO 的风险
- SSO 登录一次访问多系统，**IdP 是单点**
- IdP 被攻破→所有系统沦陷
- 需加强 IdP 安全（多因素认证）

### 易错4：生物特征不可更改
- 口令泄露可改，**生物特征泄露永久**
- 生物数据是敏感信息，需加密存储

### 易错5：CHAP 是否等于加密
- CHAP 不直接发送明文口令，使用随机挑战和响应验证共享秘密
- CHAP 不加密认证后的业务通信，机密性仍需 TLS、IPSec 等协议提供

## 6. 真题示例及解析

### 真题1（上午·单选）

> 下列哪组属于双因素认证？
> A. 口令+安全问题  B. 口令+短信验证码  C. 指纹+人脸  D. 两个口令

**答案：B**

**解析**：双因素=两个**不同类**因素。口令(知识)+短信(持有)是双因素。口令+安全问题都是知识因素，指纹+人脸都是生物因素，不算双因素。

---

### 真题2（上午·单选）

> 生物特征认证中，FAR(误识率)与FRR(拒识率)的关系是？
> A. 互相独立  B. 互相制约，降低一个会升高另一个  C. 同时升降  D. 无关

**答案：B**

**解析**：FAR(认错人，越低越安全)与FRR(认不出，越低体验越好)互相制约。调严降低FAR会升高FRR。

---

### 真题3（上午·单选）

> 单点登录(SSO)的主要安全风险是？
> A. 速度慢  B. IdP是单点，被攻破则所有系统沦陷  C. 不支持多系统  D. 无法加密

**答案：B**

**解析**：SSO登录一次访问多系统，IdP(身份提供者)是单点。IdP被攻破→所有系统沦陷。需加强IdP安全(多因素)。

---

### 真题4（上午·单选）

> Kerberos 认证协议基于的密钥体制是？
> A. 非对称  B. 对称  C. 哈希  D. 无密钥

**答案：B**

**解析**：Kerberos基于**对称密钥**+KDC第三方，用票据(Ticket)认证。PKI基于非对称+CA。

## 7. 考点速记
- 认证三因素：**知道/拥有/是**；双因素=不同类组合
- 口令存储：**哈希+盐+慢哈希(bcrypt)**
- 智能卡=**持有因素**；生物特征=**你是谁**
- 生物特征**不可更改**，泄露即永久
- SSO 登录一次访问多系统；IdP 是单点需重点防护
- **FAR** 误识率(安全)；**FRR** 拒识率(体验)；互相制约
- Kerberos 对称+KDC；PKI 非对称+CA
- PAP 直接发送口令；CHAP 挑战响应但不加密业务数据
- RADIUS 常用于网络接入；TACACS+ 常用于网络设备管理，AAA 可分离
