# 密码学应用大题

## 题型特征

通常给出一个**应用场景**（如邮件加密、文件传输、数字签名），要求：
1. 选择合适算法并说明理由
2. 进行简单计算（RSA 加解密、哈希值）
3. 分析密钥管理或安全流程

## 例题 1：RSA 计算题（高频）

> **题目**：已知 RSA 参数 p=11，q=3，e=7，明文 M=8。
> (1) 计算 n、φ(n)、d
> (2) 计算密文 C
> (3) 验证解密 M=C^d mod n

### 标准解答

**(1) 计算 n、φ(n)、d**

$$n = p \times q = 11 \times 3 = 33$$

$$\varphi(n) = (p-1)(q-1) = 10 \times 2 = 20$$

求 d：$d = e^{-1} \bmod \varphi(n)$，即 $7d \equiv 1 \pmod{20}$

试算：$7 \times 3 = 21 \equiv 1 \pmod{20}$，故 $d = 3$

> 💡 **答题技巧**：d 的计算用扩展欧几里得，但小数直接试算更快。

**(2) 计算密文 C**

$$C = M^e \bmod n = 8^7 \bmod 33$$

分步计算：
- $8^2 = 64 \equiv 31 \pmod{33}$
- $8^4 = 31^2 = 961 \equiv 4 \pmod{33}$
- $8^7 = 8^4 \times 8^2 \times 8 = 4 \times 31 \times 8 = 992 \equiv 2 \pmod{33}$

故 $C = 2$

**(3) 验证解密**

$$M = C^d \bmod n = 2^3 \bmod 33 = 8$$

M=8 与原明文一致，验证成功 ✓

> 💡 **评分要点**：n/φ/d 各 1 分，C 计算 2 分，验证 1 分，**必须写计算过程**，只有结果不给分。

## 例题 2：算法选择题

> **题目**：某系统需实现以下安全需求，请为每个需求选择最合适的算法并说明理由：
> (1) 大文件传输的机密性保护
> (2) 验证文件完整性
> (3) 发送方身份认证与不可否认
> (4) 双方协商共享密钥

### 标准解答

| 需求 | 选择算法 | 理由 |
|------|----------|------|
| (1) 大文件机密性 | **AES**（对称加密） | 对称加密速度快，适合大数据量 |
| (2) 完整性验证 | **SHA-256 / SM3**（哈希） | 哈希单向且抗碰撞，验证数据未被篡改 |
| (3) 身份认证+不可否认 | **RSA 数字签名** | 私钥签名只有持有者能签，公钥验签 |
| (4) 密钥协商 | **Diffie-Hellman** | 专为密钥协商设计，双方无需预先共享密钥 |

> 💡 **答题要点**：
> - 机密性→对称加密（AES，非 RSA，因 RSA 慢）
> - 完整性→哈希（不是加密）
> - 不可否认→数字签名（不是 MAC，MAC 无不可否认性）
> - 密钥协商→DH（不是 RSA 加密）

## 例题 3：混合加密流程

> **题目**：Alice 要给 Bob 安全发送一份机密文件，要求机密性+完整性+不可否认。请设计完整流程。

### 标准解答

```
发送方 Alice：
① 对文件 M 计算哈希：h = Hash(M)           ← 完整性
② Alice 用自己私钥签名：S = Sign(h, SK_Alice) ← 不可否认
③ 生成随机对称密钥 K，用 AES 加密 (M+S)：C = AES_K(M, S)  ← 机密性
④ 用 Bob 公钥加密对称密钥：Ck = Encrypt(K, PK_Bob) ← 安全传递密钥
⑤ 发送 (C, Ck) 给 Bob

接收方 Bob：
⑥ 用 Bob 私钥解密出 K：K = Decrypt(Ck, SK_Bob)
⑦ 用 K 解密出 (M, S)：AES解密
⑧ 对 M 重算哈希 h'，用 Alice 公钥验签 S → 对比 h'  ← 完整性+认证
```

> 💡 **评分要点**：每步 1-2 分。关键得分点：
> - ① 必须先哈希再签名（效率）
> - ③ 用对称加密文件（非 RSA，因文件大）
> - ④ 用接收方公钥传密钥（混合加密核心）
> - ⑧ 验签步骤必须写

## 答题模板速记

### 算法选择口诀
```
机密性 → 对称加密（AES/SM4）
完整性 → 哈希（SHA-256/SM3）
认证+不可否认 → 数字签名（RSA/SM2签名）
密钥协商 → DH
密钥传递 → 用接收方公钥加密对称密钥
```

### RSA 计算步骤
```
n=p×q → φ=(p-1)(q-1) → e选(与φ互素) → d=e⁻¹modφ
加密 C=M^e mod n    解密 M=C^d mod n
签名 S=H^d mod n    验签 H'=S^e mod n, 对比H'
```

## 考点速记
- RSA 计算必写过程，结果占小头
- 大文件机密性用**对称加密**（不用 RSA）
- 不可否认性只能用**数字签名**（不是 MAC）
- 混合加密 = 非对称传密钥 + 对称加密数据
- 哈希在签名前计算（对摘要签名非原文）
