# 综合分析大题

## 题型特征

综合题通常给出**复杂场景**（如某企业被攻击的全过程），要求跨模块综合分析，难度最高。考查：
1. 攻击链还原
2. 多层防护方案设计
3. 技术与管理结合

## 例题 1：攻击链分析

> **题目**：某公司发生数据泄露事件。经调查还原过程如下：
> ① 攻击者向财务员工发送含恶意附件的钓鱼邮件
> ② 员工打开附件，植入远控木马
> ③ 攻击者通过木马横向移动，获取域管理员权限
> ④ 利用域管权限访问数据库服务器，窃取数据
> ⑤ 数据外传到境外服务器
>
> 请分析每个环节对应的攻击类型和应在何处部署防护措施。

### 标准解答

| 环节 | 攻击类型 | 防护措施 |
|:----:|----------|----------|
| ① | **钓鱼攻击**（社会工程） | 安全意识培训、邮件网关过滤、SPF/DKIM/DMARC |
| ② | **恶意代码感染** | 终端杀毒、应用白名单、禁用宏、沙箱检测附件 |
| ③ | **横向移动/提权** | 最小权限、网络分段、特权账号管理、异常行为监控 |
| ④ | **越权访问** | 数据库访问控制、最小权限、审计日志 |
| ⑤ | **数据外泄** | DLP（数据防泄露）、流量监控、出口防火墙 |

> 💡 **答题技巧**：每个环节回答"攻击类型 + 对应防护"，按链路顺序逐个分析。

## 例题 2：安全方案设计

> **题目**：某电商平台需建设安全体系，包含：Web 前端、应用服务器、数据库、支付系统。请设计纵深防御方案。

### 标准解答

**纵深防御（Defense in Depth）多层防护**：

| 层次 | 防护措施 |
|------|----------|
| **网络边界** | 下一代防火墙、抗 DDoS、WAF |
| **网络传输** | HTTPS/TLS 加密、HSTS |
| **Web 应用** | 输入校验、输出编码、参数化查询、CSP |
| **应用服务器** | 最小权限、安全配置、补丁管理、日志审计 |
| **数据库** | 最小权限、透明加密 TDE、脱敏、审计 |
| **支付系统** | 双因素认证、交易签名、风控引擎、PCI DSS 合规 |
| **终端** | 杀毒、EDR、补丁 |
| **管理** | 安全制度、人员培训、应急响应、等保合规 |

**关键设计原则**：
① **纵深防御**：多层防护，单层被突破不致全盘崩溃
② **最小权限**：各组件仅授予必要权限
③ **默认安全**：默认配置安全
④ **失败安全**：异常时进入安全状态
⑤ **审计可追溯**：全链路日志

## 例题 3：等保整改方案

> **题目**：某系统等保测评发现以下问题，请给出整改方案：
> (1) 未启用安全审计
> (2) 数据库使用默认口令
> (3) 未对远程管理加密
> (4) 无入侵防范措施

### 标准解答

| 问题 | 整改措施 |
|------|----------|
| (1) 未启用审计 | ① 开启系统/应用/数据库审计日志 ② 集中日志管理 ③ 日志留存≥6个月 |
| (2) 默认口令 | ① 立即修改为强口令 ② 口令策略（长度≥8、复杂度、定期更换） ③ 失败锁定 |
| (3) 远程管理未加密 | ① 禁用 Telnet，改用 **SSH** ② 远程管理走 **VPN** ③ 限制管理 IP |
| (4) 无入侵防范 | ① 部署 **IDS/IPS** ② 主机杀毒 ③ 及时打补丁 ④ 异常流量监控 |

> 💡 **答题模板**：每个问题给出 2-3 个具体整改措施，用规范术语。

## 综合题答题策略

### 攻击链分析
```
逐环节 → 攻击类型 + 防护措施（每环节2-3分）
```

### 方案设计
```
按层次 → 网络边界/传输/应用/数据/终端/管理（每层1-2分）
+ 设计原则（纵深防御/最小权限/默认安全）
```

### 整改方案
```
逐问题 → 2-3个具体措施（用规范术语，如SSH/IDS/TDE）
```

## 高频综合考点

| 场景 | 涉及知识点 |
|------|-----------|
| APT 攻击 | 钓鱼→木马→横向移动→窃取→外传 |
| 数据泄露 | 访问控制+加密+DLP+审计 |
| 勒索病毒 | 漏洞利用→加密→备份恢复 |
| 等保整改 | 审计/口令/加密/入侵防范 |
| 云安全 | 责任共担+IAM+加密+VPC |

## 考点速记
- 综合题按**链路/层次**逐项分析，每项给"类型+防护"
- 方案设计必写**纵深防御+最小权限**
- 整改用规范术语：SSH/IDS/TDE/审计/最小权限
- 日志留存 **≥6 个月**
- 远程管理用 SSH 非 Telnet
