# 网络安全配置大题

## 题型特征

通常给出**网络拓扑图**或**抓包数据**，要求：
1. 分析防火墙规则
2. 配置 IPSec/VPN 参数
3. 识别攻击类型（基于抓包特征）
4. 设计网络安全方案

## 例题 1：防火墙规则配置

> **题目**：某企业网络拓扑：内网(192.168.1.0/24)、DMZ(Web服务器10.0.0.10)、外网。防火墙三接口。请配置 ACL 规则满足：
> (1) 外网可访问 DMZ 的 Web 服务
> (2) 内网可访问 DMZ 和外网
> (3) DMZ 不能主动访问内网

### 标准解答

| 规则 | 源地址 | 目的地址 | 端口 | 动作 | 说明 |
|:----:|--------|----------|:----:|:----:|------|
| 1 | 外网任意 | 10.0.0.10 | 80,443 | 允许 | 外网访问 DMZ Web |
| 2 | 192.168.1.0/24 | 10.0.0.0/24 | 任意 | 允许 | 内网访问 DMZ |
| 3 | 192.168.1.0/24 | 外网任意 | 任意 | 允许 | 内网访问外网 |
| 4 | 10.0.0.0/24 | 192.168.1.0/24 | 任意 | **拒绝** | DMZ 禁止访问内网 |
| 5 | 任意 | 任意 | 任意 | 拒绝 | 默认拒绝 |

> 💡 **答题要点**：
> - **默认拒绝**原则（最后一条 deny any）
> - DMZ 不能主动访问内网（防 DMZ 被攻陷后跳板）
> - 规则顺序：精确规则在前，宽泛规则在后
> - Web 服务只开 80/443，不开其他端口

## 例题 2：IPSec 配置分析

> **题目**：两个子公司通过 Internet 互联，需保护总部(A)到分公司(B)的数据传输。请说明应采用的 IPSec 模式、协议及配置。

### 标准解答

**(1) 工作模式：隧道模式（Tunnel Mode）**

理由：两个站点间通信，需保护整个 IP 包，隧道模式在两个网关间建立安全隧道，隐藏内网 IP。

**(2) 安全协议：ESP**

理由：需机密性（加密），AH 不加密只认证，ESP 既加密又认证。

**(3) 配置参数**：

| 参数 | 选择 | 理由 |
|------|------|------|
| 模式 | 隧道模式 | 站点到站点 VPN |
| 协议 | ESP | 需要加密 |
| 加密算法 | AES-256 | 安全性足够 |
| 认证算法 | SHA-256 | 完整性校验 |
| 密钥交换 | IKE/DH | 自动协商密钥 |
| DH 组 | Group 14/19 | 前向安全 |

**(4) 安全流程**：
① 两端网关通过 IKE 协商 SA（第一阶段管理 SA，第二阶段 IPSec SA）
② 内网主机 A 发送数据包到内网主机 B
③ A 端网关用 ESP 加密整个 IP 包，加新 IP 头（网关地址）发出
④ B 端网关解密，还原原始 IP 包转发给主机 B

## 例题 3：抓包识别攻击

> **题目**：入侵检测系统捕获以下异常流量，请分析攻击类型并给出防护建议。
> ```
> 大量来自不同源IP的SYN包，目标端口80，但无后续ACK完成握手
> ```

### 标准解答

**(1) 攻击类型：SYN Flood 拒绝服务攻击**

**(2) 分析依据**：
- 大量 SYN 包（发起 TCP 连接请求）
- 来自不同源 IP（分布式，可能僵尸网络 DDoS）
- 无后续 ACK（不完成三次握手）
- 目标 80 端口（攻击 Web 服务）
- 目的：耗尽服务器半连接队列，使正常用户无法连接

**(3) 防护措施**：
① **SYN Cookie**：服务器不分配半连接资源，将状态编码到 SYN-ACK 序列号
② 连接限速：限制单 IP 的 SYN 速率
③ 防火墙/IPS：识别异常 SYN 模式并阻断
④ 负载均衡/CDN：分散流量
⑤ 增加半连接队列大小（缓解，非根治）

## 答题模板：网络方案设计

```
① 需求分析：明确安全目标（机密性/完整性/可用性）
② 方案选择：选合适的协议/设备（IPSec/防火墙/IDS）
③ 参数配置：算法/模式/规则（隧道模式/ESP/AES）
④ 流程说明：数据如何流转（加密→传输→解密）
⑤ 补充措施：额外加固（默认拒绝/最小权限）
```

## 常见抓包特征-攻击类型对照

| 抓包特征 | 攻击类型 |
|---------|----------|
| 大量 SYN 无 ACK | SYN Flood |
| 大量 ICMP Echo Request | Smurf / Ping Flood |
| 单 IP 高频扫描端口 | 端口扫描 |
| DNS 响应远大于请求 | DNS 放大攻击 |
| 大量相同源目 IP 的包 | Land 攻击 |

## 考点速记
- 防火墙**默认拒绝**；DMZ 不能主动访问内网
- 站点间 VPN 用**隧道模式+ESP**
- SYN Flood 防 **SYN Cookie**
- IPSec：AH 认证不加密，ESP 加密+认证
- 抓包题先识别特征再定攻击类型
