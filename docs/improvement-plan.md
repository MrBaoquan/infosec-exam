# 大纲覆盖与内容优化计划

本页用于跟踪站点内容与信息安全工程师考试要求的对应关系。覆盖状态分为：

- **完整**：具备原理、案例、易错辨析、练习和考点速记
- **基本覆盖**：核心知识已讲解，但案例或应用题不足
- **待补强**：仅有概览、术语表或零散内容
- **待核验**：需依据现行官方考试大纲确认范围和深度

> 大纲与考试安排可能调整。考试形式、时间和范围以[中国计算机技术职业资格网](https://www.ruankao.org.cn/)发布的信息为准。

## 教程质量标准

每篇主干知识文档按以下顺序编写：

1. **为什么需要**：从业务场景和待解决问题切入
2. **是什么**：给出定义、全称和形象比喻
3. **怎么工作**：解释流程、参与者和关键机制
4. **具体例子**：提供配置、计算或真实业务场景
5. **易错辨析**：比较相近概念和常见错误选项
6. **真题风格练习**：至少一道上午单选和一道下午简答或小案例
7. **考点速记**：只保留经过正文解释的结论

## 章节组织原则

- 全站遵循“基础理论 → 技术原理 → 攻击威胁 → 防护措施 → 管理实践”。
- 同一知识点只保留一个主讲页面，其他页面使用链接引用。
- 协议归入网络或认证章节，密码模式归入密码学章节，工程活动归入生命周期章节。
- 前沿内容先给出完整场景，再介绍缩写、框架和工具，避免术语连续堆叠。
- 学习路线必须覆盖侧边栏全部主干文章；低频补充内容可标记为选学，但不能静默遗漏。

- [ ] 每篇主干文档增加下午简答或场景小题

- [ ] 每次大纲或法规更新后执行覆盖复核

## 验收规则

一次内容优化只有同时满足以下条件才算完成：

1. 大纲矩阵状态已更新；
2. 文档达到对应教程质量标准；
3. 学习路线和侧边栏顺序一致；
4. 相关题目解析说明正确答案和错误选项原因；
5. 375px 窄屏内容不溢出；
6. `npm run build` 构建成功。
