# 一、信息安全基础

## CIA 三性（核心）

- **机密性** Confidentiality — 防泄露
- **完整性** Integrity — 防篡改
- **可用性** Availability — 防中断

扩展：认证性、不可否认性、可控性、可审计性。

## 安全模型

| 模型 | 关注 | 特点 |
|------|------|------|
| **BLP** | 机密性 | "向下读、向上写"，防高密级信息流向低密级 |
| **Biba** | 完整性 | "向上读、向下写"，防低完整性数据污染高完整性 |
| **Clark-Wilson** | 完整性 | 职责分离 + 事务转换 |
| **Chinese Wall** | 利益冲突 | 防止访问有利益冲突的数据 |

> 🔑 BLP 与 Biba 方向**正好相反**，是高频考点。

## 安全等级

- **TCSEC**（橘皮书）四组七级：D < C1 < C2 < B1 < B2 < B3 < A1
- **CC** 通用准则 EAL1~EAL7
- **等保 2.0**（中国）五级：1自主保护 ~ 5专控保护

## 法律法规（高频）

- 《网络安全法》2017 施行 — 网络运营者责任、关键信息基础设施保护
- 《数据安全法》2021.9
- 《个人信息保护法》2021.11 — 最小必要、告知同意
- 《密码法》2020.1 — 核心密码/普通密码/商用密码分类

## 标准化

- ISO/IEC 27001 信息安全管理体系 ISMS
- ISO/IEC 27002 控制措施
- GB/T 22239 等保 2.0 基本要求

## 考点速记

- CIA 三性是安全目标基石
- BLP 保机密性（下读上写），Biba 保完整性（上读下写）
- 等保 2.0 五级，三级及以上需测评
- 三部核心法：网络安全法 / 数据安全法 / 个人信息保护法
