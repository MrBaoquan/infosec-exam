# 信息安全基本概念

## 信息安全定义

信息安全是指保护信息的**机密性、完整性、可用性**，即 CIA 三性，防止信息被未授权访问、篡改、破坏。

> 扩展：认证性、不可否认性、可控性、可审计性。

## CIA 三性（核心考点）

| 属性 | 含义 | 对应威胁 | 典型防护 |
|------|------|----------|----------|
| **机密性** Confidentiality | 信息不被未授权获取 | 窃听、泄露 | 加密、访问控制 |
| **完整性** Integrity | 信息不被未授权篡改 | 篡改、伪造 | 哈希、数字签名 |
| **可用性** Availability | 信息和系统可被授权者及时访问 | DoS、破坏 | 冗余、备份、负载均衡 |

### 三性易错辨析
- 加密→机密性（不是完整性）
- 哈希→完整性（不是机密性，哈希不加密）
- 备份→可用性（不是机密性）
- 数字签名→完整性+认证+不可否认

## 扩展安全属性

| 属性 | 含义 |
|------|------|
| **认证性** | 实体身份可验证（你是谁） |
| **不可否认性** | 行为不可抵赖（不能否认做过） |
| **可控性** | 对信息流向可控制 |
| **可审计性** | 行为可被记录追溯 |

## 信息安全威胁分类

### 按攻击方式
| 类型 | 特征 | 代表 |
|------|------|------|
| **被动攻击** | 只窃听不修改，难发现 | 嗅探、流量分析 |
| **主动攻击** | 篡改/伪造/中断 | 篡改、伪造、DoS、重放 |

> 被动攻击防：**加密**；主动攻击防：**认证+完整性+访问控制**

### 按安全属性破坏
- 机密性威胁：窃听、泄露
- 完整性威胁：篡改、伪造
- 可用性威胁：DoS、破坏
- 认证性威胁：伪装、重放

## 安全服务与机制（OSI/X.800）

| 安全服务 | 对应机制 |
|---------|----------|
| 认证 | 数字签名、口令、生物特征 |
| 访问控制 | ACL、防火墙 |
| 机密性 | 加密 |
| 完整性 | 哈希、MAC |
| 不可否认 | 数字签名、审计 |

## 考点速记
- **CIA = 机密性/完整性/可用性**
- 加密→机密性；哈希→完整性；备份→可用性
- 被动攻击防加密；主动攻击防认证+完整性
- 数字签名提供完整性+认证+不可否认（**不提供机密性**）
- 不可否认性只能用数字签名实现（MAC 不能）
