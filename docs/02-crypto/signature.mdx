# 数字签名

## 核心目的

提供 **完整性 + 认证性 + 不可否认性**（注意：**不提供机密性**）。

## 基本流程

```
发送方：摘要 = H(消息)  →  签名 = 摘要 ^ 私钥  →  发送 (消息, 签名)
接收方：摘要' = H(消息)  →  验签 = 签名 ^ 公钥  →  对比 摘要' 是否相等
```

- **私钥签名**（只有持有者能签 → 不可否认）
- **公钥验签**（任何人可验 → 认证）
- 对**摘要**签名而非原文（效率，原文可能很大）

## 与公钥加密的对比（高频易错点）

| 操作 | 用谁的密钥 |
|------|-----------|
| 公钥**加密** | 接收方公钥 |
| 私钥**解密** | 接收方私钥 |
| 私钥**签名** | 发送方私钥 |
| 公钥**验签** | 发送方公钥 |

> ⚠️ 易错：加密用**接收方**的密钥对；签名用**发送方**的密钥对。

## 常见签名算法

| 算法 | 基础 | 说明 |
|------|------|------|
| RSA 签名 | 大整数分解 | 最经典，PKCS#1 |
| DSA | 离散对数 | 仅签名不能加密 |
| ECDSA | 椭圆曲线 | 比特币/区块链常用 |
| ElGamal 签名 | 离散对数 | DSA 的原型 |
| SM2 签名 | 椭圆曲线 | 国密签名标准 |

## 同时保证机密性 + 完整性 + 不可否认

**先签名后加密**（推荐）：
1. A 用 A 的私钥对消息签名
2. A 用 B 的公钥加密（消息 + 签名）
3. B 用 B 的私钥解密
4. B 用 A 的公钥验签

## 考点速记

- 数字签名提供 **完整性+认证+不可否认**，**不提供机密性**
- 私钥签、公钥验；用**发送方**的密钥对
- 对**消息摘要**签名（非原文）
- DSA 只能签名不能加密；ECDSA 基于椭圆曲线
- 同时保密+签名：**先签后密**
