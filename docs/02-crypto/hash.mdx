# 哈希函数（散列/消息摘要）

## 定义与性质

将**任意长度**输入压缩为**定长**输出（摘要），是**单向不可逆**的。

> 哈希**不是加密**！加密可解密还原，哈希不可逆。哈希用于**完整性校验**。

## 三大安全性质（必背）

| 性质 | 含义 |
|------|------|
| **单向性** | 由 H 找不到 M（前置像抗性） |
| **抗弱碰撞** | 给定 M₁，难找 M₂≠M₁ 使 H(M₁)=H(M₂)（第二原像抗性） |
| **抗强碰撞** | 难找任意 M₁≠M₂ 使 H(M₁)=H(M₂)（碰撞抗性，最难满足） |

> 强碰撞 ⟹ 弱碰撞 ⟹ 单向性（强度递减）

## 常见哈希算法

| 算法 | 输出位数 | 状态 |
|------|----------|------|
| MD4 | 128 | 已破 |
| **MD5** | 128 | **已破**（可构造碰撞，不应用于安全场景） |
| **SHA-1** | 160 | **已破**（2017 Google 碰撞） |
| SHA-256 | 256 | 安全（SHA-2 家族） |
| SHA-512 | 512 | 安全 |
| SHA-3 | 224~512 | 安全（Keccak，海绵结构） |
| **SM3** | 256 | 国密，安全 |

## MAC 与 HMAC

- **MAC**（消息认证码）= 带密钥的哈希，同时提供**完整性 + 认证**
- **HMAC** = Hash-based MAC，用现有哈希构造（如 HMAC-SHA256）
- 哈希无密钥 → 只能验证完整性；MAC 有密钥 → 还能验证来源

## 消息认证 vs 数字签名

| 对比项 | MAC | 数字签名 |
|--------|-----|----------|
| 密钥 | 对称共享密钥 | 非对称私钥签/公钥验 |
| 不可否认 | ❌（双方都有密钥，可抵赖） | ✅（仅私钥持有者能签） |
| 速度 | 快 | 慢 |

## 考点速记

- 哈希**单向**、**定长输出**、**抗碰撞**
- 三性质强度：强碰撞 > 弱碰撞 > 单向
- **MD5/SHA-1 已破**，新系统用 SHA-256/SM3
- 哈希无密钥只验完整性；MAC 有密钥还能验来源；签名有不可否认性
- 雪崩效应 = 输入微小变化→输出剧变（约 50% bit 翻转）
