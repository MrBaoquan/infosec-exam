# 对称加密：DES 与 3DES

## DES 基本参数

| 参数 | 值 |
|------|-----|
| 分组长度 | **64 位** |
| 密钥长度 | 64 位（含 8 位奇偶校验）→ **有效 56 位** |
| 轮数 | **16 轮** |
| 结构 | **Feistel 网络** |

## Feistel 网络核心思想

- 64 位明文经**初始置换 IP** → 左 32 位 L₀ + 右 32 位 R₀
- 16 轮迭代，每轮：
  - $L_i = R_{i-1}$
  - $R_i = L_{i-1} \oplus f(R_{i-1}, K_i)$
- 最后 **L₁₆ 与 R₁₆ 交换**，再经**逆初始置换 IP⁻¹** → 密文

> 🔑 Feistel 的精妙：**加解密结构完全相同**，解密只需把子密钥**逆序**使用（K₁₆→K₁）。

## 轮函数 f 的内部结构

```
R(32位) → 扩展置换 E(32→48位)
        → 与子密钥 Ki(48位) 异或
        → 进入 8 个 S 盒(48→32位，核心非线性)
        → P 盒置换(32→32位)
        → 输出 f(R,Ki)
```

- **S 盒**是 DES 唯一非线性部件，6 位输入→4 位输出，共 8 个
- 子密钥生成：64 位主密钥 → PC-1 选 56 位 → C/D 分半 → 循环左移 → PC-2 选 48 位

## 3DES（三重 DES）

- 用 2 或 3 个密钥，DES 调用三次：**加密-解密-加密**
  - 2 密钥：$C = E_{K1}(D_{K2}(E_{K1}(M)))$ → 有效 112 位
  - 3 密钥：$C = E_{K3}(D_{K2}(E_{K1}(M)))$ → 有效 168 位
- 中间用解密是为兼容单 DES（K1=K2=K3 时退化为 DES）
- 缓解 DES 密钥过短问题，但速度慢，逐渐被 AES 取代

## DES 安全性

- **有效密钥仅 56 位**，1998 年 EFF 机器 22 小时破解 → 已不安全
- 差分密码分析、线性密码分析对其有效（虽需大量明密文对）
- 不再用于新系统，遗留系统用 3DES 过渡

## 考点速记

- DES 分组 **64 位**，有效密钥 **56 位**，**16 轮**，Feistel 结构
- Feistel **加解密同结构，子密钥逆序**
- 轮函数 f = 扩展E → 异或Ki → **8个S盒** → P盒
- 3DES 中间步骤是**解密**（为兼容单 DES）
- DES 不安全主因：**密钥太短(56位)**
