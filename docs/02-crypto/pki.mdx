# PKI / CA / 数字证书

## PKI 五大组件

| 组件                 | 作用                   |
| -------------------- | ---------------------- |
| **CA** 证书颁发机构  | 签发/管理证书（核心）  |
| **RA** 注册机构      | 受理申请，审核身份     |
| **证书库**           | 存储已签发证书，供查询 |
| **CRL** 证书吊销列表 | 作废证书清单           |
| **端实体**           | 持证用户/应用          |

## 数字证书（X.509）

绑定**公钥**与**身份**，由 CA 用其私钥签名，解决"公钥真实性"问题（防中间人）。

证书内容：

- 版本、序列号、签名算法
- 持有者名称、**持有者公钥**
- 颁发者(CA)名称、有效期
- **CA 的数字签名**

## 信任链与验证

- 终端证书 → 中间 CA → **根 CA（自签名，预置信任）**
- 验证：逐级验签直至根 CA，并检查有效期 + CRL/OCSP 是否吊销

## 证书吊销方式

| 方式     | 机制             | 特点         |
| -------- | ---------------- | ------------ |
| **CRL**  | 定期发布吊销列表 | 离线、有延迟 |
| **OCSP** | 在线实时查询状态 | 实时，常用   |

## 常见证书类型

- **DV** 域名验证（只验域名）
- **OV** 组织验证（验企业身份）
- **EV** 扩展验证（最严格，浏览器显示组织名）

## HTTPS 中的应用

1. 客户端连服务器，服务器返回证书（含公钥）
2. 客户端用 CA 公钥验签证书 → 确认服务器身份
3. 协商出对称会话密钥（如 ECDHE）
4. 用对称密钥加密通信（TLS 记录层）

## 5. 易错辨析

### 易错1：CA用谁的密钥签证书
- CA用**自己的私钥**签发证书
- 用户用**CA的公钥**验签证书
- 不是用用户的密钥

### 易错2：证书吊销方式
- **CRL**：离线吊销列表，有延迟
- **OCSP**：在线实时查询，常用

### 易错3：信任链
- 终端证书→中间CA→根CA(自签名)
- 根CA是**自签名**，预置在操作系统/浏览器中

## 6. 真题示例及解析

### 真题1（上午·单选）

> PKI 中，CA 签发数字证书使用的是？
> A. 用户的公钥  B. CA自己的私钥  C. 用户的私钥  D. CA自己的公钥

**答案：B**

**解析**：CA用自己的私钥对证书签名，用户用CA的公钥验签。证书绑定的是"用户身份+用户公钥"。

---

### 真题2（上午·单选）

> 实时查询证书是否被吊销的协议是？
> A. CRL  B. OCSP  C. LDAP  D. SNMP

**答案：B**

**解析**：OCSP在线实时查询。CRL是离线吊销列表有延迟。

---

### 真题3（上午·单选）

> PKI 信任链的根是？
> A. 中间CA  B. 根CA(自签名)  C. 用户  D. RA

**答案：B**

**解析**：根CA是自签名证书，预置信任。逐级验签至根CA。

## 7. 考点速记

- PKI 核心 = **CA**；证书 = CA 签名的"公钥+身份"绑定
- 证书格式 **X.509**；信任链止于**自签名根 CA**
- 吊销：**CRL**（离线列表）/ **OCSP**（在线实时）
- CA 用**自己的私钥**给证书签名
- HTTPS = TLS = 非对称协商密钥 + 对称加密数据 + 证书验身份
