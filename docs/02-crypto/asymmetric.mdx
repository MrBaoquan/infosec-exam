# 非对称加密：RSA

## 非对称加密总览

| 算法 | 数学难题 | 用途 |
|------|----------|------|
| **RSA** | 大整数分解 | 加密/签名/密钥交换 |
| **ECC** | 椭圆曲线离散对数 | 签名/密钥交换(短密钥) |
| **ElGamal** | 离散对数 | 加密/签名 |
| **DH** | 离散对数 | 密钥协商(不能加密) |
| **SM2** | 椭圆曲线 | 国密公钥加密/签名 |

> 特点：**公钥加密/私钥解密**；速度远慢于对称加密（约慢 100~1000 倍）；
> 实践中常**混合加密**：非对称加密对称密钥，对称加密数据。

## RSA 算法步骤

1. 选大素数 $p, q$
2. $n = p \times q$
3. $\varphi(n) = (p-1)(q-1)$（欧拉函数）
4. 选公钥 $e$，满足 $1 < e < \varphi(n)$ 且 $\gcd(e, \varphi(n)) = 1$
5. 计算私钥 $d = e^{-1} \bmod \varphi(n)$（扩展欧几里得算法）
6. 公钥 $(e, n)$，私钥 $(d, n)$
7. 加密：$C = M^e \bmod n$
8. 解密：$M = C^d \bmod n$

## RSA 安全性

- 基于**大整数分解难题**：已知 $n$ 难以分解出 $p, q$
- 安全密钥长度：**≥ 2048 位**（1024 位已不推荐）
- 选择密文攻击、共模攻击、低指数攻击是经典考点

## RSA 数字签名

- 签名：$S = M^d \bmod n$（**用私钥**）
- 验签：$M = S^e \bmod n$（用公钥）
- 与加密**方向相反**：私钥签、公钥验 → 实现**不可否认性**

## Diffie-Hellman 密钥交换（高频）

- Alice: 选 $a$，发 $A = g^a \bmod p$
- Bob: 选 $b$，发 $B = g^b \bmod p$
- 共享密钥：$K = B^a = A^b = g^{ab} \bmod p$
- 离散对数难解 → 攻击者无法由 $A,B$ 求 $a,b$
- ⚠️ **无法认证身份**，易受**中间人攻击**（需结合数字签名/证书）

## 考点速记

- RSA 基于**大整数分解**，ECC 基于**椭圆曲线离散对数**
- RSA 公钥加密、私钥解密；签名则**私钥签、公钥验**
- RSA 速度慢 → **混合加密**（非对称传密钥 + 对称传数据）
- DH 用于**密钥协商**，不加密、不认证，易遭**中间人攻击**
- ECC 密钥短（256 位 ≈ RSA 3072 位安全性），适合移动端
