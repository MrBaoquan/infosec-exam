# 网络安全测评技术与标准

> 🎯 **形象比喻**：安全测评像"房屋验收"——检查结构(技术)、管理(制度)、抗压(渗透)是否达标。

## 测评概念

依据标准对信息系统的**安全状况**进行检测、评估，给出符合性结论。

## 测评类型（4种分类维度）

### 按目标分
| 类型 | 说明 |
|------|------|
| **等保测评** | 是否符合等级保护要求 |
| **验收测评** | 系统上线前安全验收 |
| **风险测评** | 评估安全风险等级 |

### 按内容分
| 类型 | 说明 |
|------|------|
| **技术安全测评** | 物理环境/通信网络/计算环境/管理中心 |
| **管理安全测评** | 制度/机构/人员/建设/运维 |

### 按实施方式分
| 类型 | 说明 |
|------|------|
| **安全功能检测** | 验证安全功能是否正常 |
| **安全管理检测** | 查制度/流程 |
| **代码安全审查** | 源码/二进制审查 |
| **网络安全渗透** | 模拟攻击找漏洞 |
| **信息系统攻击测试** | 真实攻击测试 |

### 按保密性分
- 涉密信息系统测评 / 非涉密信息系统测评

## 等保测评流程

```
1. 测评准备 → 2. 方案编制 → 3. 现场测评 → 4. 分析研判 → 5. 报告编制
```

### 技术安全测评5大内容（三重防护+中心）
| 维度 | 内容 |
|------|------|
| **安全物理环境** | 机房/供电/防盗 |
| **安全通信网络** | 网络架构/通信加密 |
| **安全区域边界** | 边界防护/访问控制/入侵防范 |
| **安全计算环境** | 身份鉴别/访问控制/审计/恶意代码防范 |
| **安全管理中心** | 集中管理/集中审计/集中管控 |

### 管理安全测评5大内容
| 维度 | 内容 |
|------|------|
| **安全管理制度** | 总纲/细则/操作规程 |
| **安全管理机构** | 岗位/授权/沟通 |
| **安全人员管理** | 入职/培训/离职 |
| **安全建设管理** | 定级/设计/实施/验收 |
| **安全运维管理** | 环境/资产/介质/漏洞/应急 |

## 渗透测试5阶段（高频考点）

> 🎯 **形象比喻**：渗透测试像"合法的入室盗窃演练"——经授权，找弱点，出报告，不破坏。

```
① 委托受理 → ② 准备 → ③ 实施 → ④ 综合评估 → ⑤ 结题
```

| 阶段 | 关键工作 |
|:----:|----------|
| **① 委托受理** | 明确范围、授权、签合同 |
| **② 准备** | 制定方案、工具准备、风险评估 |
| **③ 实施** | 信息收集→漏洞扫描→漏洞利用→权限维持 |
| **④ 综合评估** | 风险分析、影响评估、修复建议 |
| **⑤ 结题** | 出报告、交付、复测 |

### 渗透测试内容
1. **信息收集**：目标资产、网络拓扑、技术栈
2. **漏洞扫描及发现**：自动扫描+人工挖掘
3. **漏洞分析及利用验证**：确认可利用性
4. **漏洞风险处置与修补**：给出修复方案

### 渗透测试分类
| 类型 | 说明 |
|------|------|
| **黑盒** | 不知内部，模拟外部攻击 |
| **白盒** | 完全知情，模拟内部/深度测试 |
| **灰盒** | 部分知情 |

## 测评工具

| 类别 | 工具 |
|------|------|
| 漏洞扫描 | Nessus、AWVS、Nexpose |
| 协议分析 | TCPDump、Wireshark |
| 性能测试 | Apache Meter(JMeter) |
| 代码审查 | SonarQube、Fortify |

## 测评标准

| 标准 | 说明 |
|------|------|
| 等保测评标准 | GB/T 28448 |
| 产品测评标准 | GB/T 18336(等同CC) |
| 风险评估标准 | GB/T 20984 |
| 密码应用安全 | GM/T 0054 |
| 工控安全评估 | GB/T 32919 |

## 考点速记
- 测评5阶段：**委托受理→准备→实施→综合评估→结题**
- 技术测评5内容：**物理环境/通信网络/区域边界/计算环境/管理中心**
- 管理测评5内容：**制度/机构/人员/建设/运维**
- 渗透测试：黑盒(不知)/白盒(全知)/灰盒(部分)
- 等保测评标准 **GB/T 28448**
- 渗透测试与黑客区别：**经授权、不破坏、出报告**
