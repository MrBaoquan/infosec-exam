# 入侵检测系统 IDS 与入侵防御系统 IPS

> 🎯 **形象比喻**：
> - **IDS**(Intrusion Detection System) = **监控摄像头**（只看只报警，不抓人，旁路部署）
> - **IPS**(Intrusion Prevention System) = **保安**（当场拦截，串联部署）

## IDS 入侵检测系统

- **旁路部署**（镜像流量），**仅告警不阻断**
- 作用：发现可疑行为、记录证据、告警

### 检测方法对比

| 方法 | 原理 | 能检已知攻击 | 能检未知攻击 | 误报率 |
|------|------|:----------:|:----------:|:------:|
| **误用检测**（特征） | 特征库匹配 | ✅ | ❌ | 低 |
| **异常检测**（行为） | 偏离基线即报警 | ❌ | ✅ | 高 |
| 混合检测 | 两者结合 | ✅ | 部分 | 中 |

> 误用检测=专家系统/特征签名，像"病毒库"；异常检测=统计学/机器学习，像"行为基线"。

### IDS 分类（按数据源）

| 类型 | 监控对象 | 部署 |
|------|----------|------|
| **NIDS** 网络型 | 网络流量 | 关键网段旁路 |
| **HIDS** 主机型 | 系统日志/文件/进程 | 各主机安装代理 |
| **DIDS** 分布式 | 多点汇总分析 | 集中控制台 |

## IPS 入侵防御系统

- **串联部署**（在线），**可阻断**
- 与 IDS 区别：**串联+阻断** vs **旁路+告警**
- IPS 必须实时处理，不能影响正常流量（性能要求高）

| 对比 | IDS | IPS |
|------|-----|-----|
| 部署 | 旁路 | 串联 |
| 响应 | 告警 | 阻断 |
| 实时性 | 准实时 | 实时 |
| 误报影响 | 仅误报 | 误报会中断正常业务 |

## 蜜罐 Honeypot

- 伪装的"靶机"，故意暴露漏洞吸引攻击
- 价值：**收集攻击手法**、转移攻击者注意力、早期预警
- 类型：低交互（模拟服务）vs 高交互（真实系统）

## 考点速记

- IDS 旁路仅告警；IPS 串联可阻断
- 误用检测=特征=已知攻击（低误报）；异常检测=基线=未知攻击（高误报）
- NIDS 看流量，HIDS 看主机日志
- 蜜罐用于**收集攻击信息**，是主动防御
- IPS 误报会中断业务，故需谨慎配置
