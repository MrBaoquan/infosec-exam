# SSL/TLS 与 HTTPS

## 演进历史

SSL 1.0/2.0/3.0（网景）→ TLS 1.0/1.1/**1.2**/**1.3**

> SSL 各版本已废弃，现代使用 **TLS 1.2/1.3**。常说"SSL 证书"实指 TLS 证书。

## TLS 握手流程（TLS 1.2 简化）

```
Client                              Server
  |  ① ClientHello（支持的密码套件、随机数）  →
  |  ② ServerHello（选定套件、随机数、证书）  ←
  |  ③ 客户端验证证书（CA 签名、有效期、域名）
  |  ④ 客户端生成预主密钥，用服务器公钥加密发送  →
  |  ⑤ 双方用三个随机数生成会话密钥
  |  ⑥ 切换到加密通信（对称加密）
```

### TLS 的安全目标
1. **身份认证**：服务器证书（防中间人）
2. **密钥协商**：安全交换对称密钥（非对称）
3. **加密传输**：对称加密保护数据
4. **完整性**：MAC/HMAC 防篡改

## 密码套件命名

格式：`密钥交换_认证_加密_哈希`

例：`ECDHE-RSA-AES256-GCM-SHA384`
- ECDHE：密钥交换（临时椭圆曲线 DH）
- RSA：服务器认证（证书签名算法）
- AES256-GCM：对称加密（AES-256-GCM 模式自带认证）
- SHA384：哈希

## HTTPS = HTTP over TLS

- 默认端口 **443**（HTTP 为 80）
- 流程：TCP 握手 → TLS 握手 → HTTP 通信（加密）

## TLS 1.3 改进

- 握手从 2-RTT 减到 **1-RTT**（支持 0-RTT 恢复）
- **强制完美前向安全 PFS**（禁用 RSA 密钥交换）
- 精简密码套件，移除不安全算法
- 禁用压缩（防 CRIME 攻击）

## 常见 TLS 攻击

| 攻击 | 原理 | 防御 |
|------|------|------|
| **降级攻击** | 强制使用弱版本/弱算法 | 强制最高版本、禁弱套件 |
| **中间人攻击** | 伪造证书截获 | 严格证书验证、HSTS |
| **BEAST/CRIME/POODLE** | 针对旧 SSL/TLS | 升级到 TLS 1.2+ |
| **Heartbleed** | OpenSSL 缓冲区溢出 | 打补丁、禁用旧版 |

## 5. 易错辨析

### 易错1：TLS 握手用什么协商密钥
- **非对称**加密协商**对称**会话密钥
- 对称密钥用于后续数据加密
- 证书用于**身份认证**（防中间人）
- 三者缺一不可

### 易错2：HTTPS 端口
- HTTP = **80**
- HTTPS = **443**
- 不是8080或8443

### 易错3：TLS 1.3 vs 1.2
- TLS 1.3：**1-RTT**握手、强制**PFS**、精简套件、0-RTT恢复
- TLS 1.2：2-RTT、允许RSA密钥交换(无PFS)
- PFS(前向安全)：长期密钥泄露不影响历史会话

## 6. 真题示例及解析

### 真题1（上午·单选）

> HTTPS 默认使用的端口是？
> A. 80  B. 443  C. 8080  D. 8443

**答案：B**

**解析**：HTTPS = HTTP over TLS，默认端口**443**。HTTP默认80。8080/8443是常用非标准端口。

---

### 真题2（上午·单选）

> TLS 握手过程中，客户端验证服务器身份使用的是？
> A. 客户端私钥  B. 服务器证书(CA签名)  C. 对称密钥  D. 哈希值

**答案：B**

**解析**：服务器发送证书，客户端用CA公钥验签证书，确认服务器身份。防中间人攻击。非对称协商密钥，对称加密数据。

---

### 真题3（上午·单选）

> TLS 1.3 相比 TLS 1.2 的改进不包括？
> A. 握手减到1-RTT  B. 强制PFS  C. 精简密码套件  D. 支持RSA密钥交换

**答案：D**

**解析**：TLS 1.3 **禁用RSA密钥交换**（强制PFS），只允许ECDHE等临时密钥交换。ABC都是1.3改进。

---

### 真题4（上午·单选）

> TLS 的安全目标不包括？
> A. 身份认证  B. 机密性  C. 完整性  D. 不可否认性

**答案：D**

**解析**：TLS提供身份认证(证书)+机密性(加密)+完整性(MAC)。**不提供不可否认性**——双方都有会话密钥，无法区分谁发的。不可否认需数字签名。

## 7. 考点速记

- SSL 已废弃，用 TLS 1.2/1.3
- TLS 握手：**非对称协商密钥 + 对称加密数据 + 证书验身份**
- HTTPS 端口 443；HTTP 80
- TLS 1.3：1-RTT、强制 PFS、精简套件
- 密码套件格式：`密钥交换_认证_加密_哈希`
- 降级攻击防御：禁用弱版本/弱算法
