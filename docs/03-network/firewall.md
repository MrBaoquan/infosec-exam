# 防火墙技术

## 防火墙基本概念

防火墙是位于可信网络与不可信网络之间的**访问控制设备**，遵循**默认拒绝**原则。

> 核心功能：控制进出网络的流量，而非检测内部攻击。

## 防火墙类型（按技术）

| 类型 | 工作层次 | 特点 | 缺点 |
|------|----------|------|------|
| **包过滤** | 网络层 | 检查 IP/端口/协议，速度快 | 不检状态、不检应用层、无法防 IP 欺骗 |
| **状态检测** | 网络/传输 | 维护连接状态表，主流 | 无法深入应用层 |
| **应用代理** | 应用层 | 完全理解应用协议，安全性最高 | 性能低、需为每个应用单独代理 |
| **电路级网关** | 会话层 | 不检查应用内容，只验证连接 | 安全性低于应用代理 |
| **NGFW 下一代防火墙** | 多层 | 集成 IDS/IPS、应用识别、深度包检测 | 成本高 |

## 防火墙部署方式

| 模式 | 说明 | 适用 |
|------|------|------|
| **透明模式** | 防火墙如网桥，不改变网络拓扑 | 旁路部署、无需改 IP |
| **路由模式** | 防火墙作路由器，跨网段 | 常见企业部署 |
| **混合模式** | 部分接口透明、部分路由 | 复杂环境 |

## DMZ 非军事区

- 位于内外网之间的缓冲区，部署对外服务（Web、邮件、DNS）
- 规则：内网可访问 DMZ；DMZ **不能主动访问**内网；外网只能访问 DMZ
- 实现：三接口防火墙（内网/外网/DMZ）

## 防火墙局限性

1. **防外不防内**：无法防御内部攻击
2. **无法绕过**：加密流量（如 VPN 内）可能绕过应用层检查
3. **单点故障**：防火墙自身宕机则全网中断
4. **不能防社会工程**：人为泄露绕过技术防护
5. **不能防未连接攻击**：如通过 U盘 传播

## 考点速记

- 包过滤看 IP/端口（快但不检状态）；状态检测维护连接表（主流）
- 应用代理最安全但最慢；NGFW 集成 IPS/应用识别
- DMZ 规则：**DMZ 不能主动访问内网**
- 防火墙默认拒绝原则
- 防火墙**防外不防内**，不能防社会工程
