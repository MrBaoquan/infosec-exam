# 网络物理隔离技术

> 🎯 **形象比喻**：物理隔离像"护城河"——内外网之间完全断开，不是防火墙的"门卫"（逻辑隔离），而是物理上不连通。

## 物理隔离 vs 逻辑隔离

| 对比 | 物理隔离 | 逻辑隔离 |
|------|----------|----------|
| 连接 | **物理断开** | 有连接但受控 |
| 安全性 | 最高 | 中等 |
| 数据交换 | 需特殊机制 | 可直接通信 |
| 代表 | 网闸、双硬盘 | 防火墙、VLAN |

## 物理隔离系统类型

| 类型 | 说明 |
|------|------|
| **双向物理隔离** | 两网可互相交换数据（通过网闸） |
| **单向物理隔离** | 数据只能单向流动（如只能从低密级到高密级） |
| **终端物理隔离** | 单台终端在不同时刻连接不同网络 |

## 物理隔离实现技术

### 终端级隔离
| 技术 | 说明 |
|------|------|
| **双PC** | 两台电脑分别连内外网，物理上完全分开 |
| **单硬盘内外分区** | 一块硬盘分两个隔离分区，切换时重启 |
| **双硬盘** | 两块硬盘分别对应内外网，切换时切换硬盘 |

### 网络级隔离
| 技术 | 说明 |
|------|------|
| **网闸(Gap)** | 内外网之间硬件开关切换，任一时刻只连一端 |
| **信息摆渡** | 数据像"摆渡船"在两网间传递，先存再转 |
| **单向传输部件** | 物理单向光闸，数据只能单向流动 |
| **协议隔离** | 不用TCP/IP，用私有协议交换 |

## 网闸工作原理（高频）

```
内网 ←→ [网闸] ←→ 外网
         ↑
    硬电子开关
    任一时刻只连通一端
```

1. 网闸从内网读取数据，存入内部存储
2. **断开**内网连接，**接通**外网
3. 将数据写入外网
4. **断开**外网，循环往复

> 🔑 **核心**：网闸在任一时刻只与一端连接，**物理上不存在同时连通的通路**，因此无法被网络攻击穿透。

## 信息摆渡

- 类似"摆渡船"：数据从A岸装船→船到B岸卸载
- 船（隔离区）任一时刻只在一岸
- 适合需要数据交换但需物理隔离的场景

## 物理隔离应用场景

- **涉密网与互联网隔离**：政府/军工核心网
- **生产网与办公网隔离**：工控系统（SCADA）
- **业务网与互联网隔离**：金融核心系统

## 5. 易错辨析

### 易错1：物理隔离 vs 逻辑隔离
- **物理隔离**：物理断开(网闸/双PC)
- **逻辑隔离**：有连接受控(防火墙/VLAN)

### 易错2：网闸核心原理
- 硬件开关切换，**任一时刻只连一端**
- 无法被网络攻击穿透

### 易错3：信息摆渡
- 数据像"摆渡船"传递，先存再转
- 隔离区任一时刻只在一岸

## 6. 真题示例及解析

### 真题1（上午·单选）

> 网闸(Gap)能防网络攻击穿透的核心原理是？
> A. 加密传输  B. 任一时刻只连通一端  C. 防火墙过滤  D. VLAN隔离

**答案：B**

**解析**：网闸硬件开关切换，任一时刻只与一端连接，物理上不存在同时连通的通路。

---

### 真题2（上午·单选）

> 下列属于物理隔离的是？
> A. 防火墙  B. VLAN  C. 网闸  D. 访问控制列表

**答案：C**

**解析**：网闸是物理隔离(物理断开)。防火墙/VLAN/ACL都是逻辑隔离(有连接受控)。

## 7. 考点速记
- 物理隔离=**物理断开**；逻辑隔离=防火墙/VLAN（有连接受控）
- **网闸**：硬件开关切换，任一时刻只连一端，无法穿透
- **信息摆渡**：数据像摆渡船传递，先存再转
- 单向传输：数据只能单向流（低→高密级）
- 涉密介质交换需通过**网闸/摆渡**，不能直接联网
- 终端隔离：双PC最安全，单硬盘分区需重启切换
