# 现代安全架构与运营

## 1. 为什么边界防火墙不再够用

某员工在家中通过云应用办公，攻击者利用钓鱼邮件窃取其账号，再使用合法账号登录企业系统。这个过程中，流量可能没有穿过传统办公网边界，也不一定包含已知恶意特征。

现代安全架构需要连续回答四个问题：

1. **谁可以访问什么**：零信任和身份访问控制；
2. **攻击者正在做什么**：Kill Chain 和 ATT&CK；
3. **如何汇总证据并发现异常**：EDR、NDR 和 SIEM；
4. **谁来调查并处置**：SOC、SOAR 和事件响应流程。

> 🎯 **整体理解**：零信任负责缩小可访问范围，ATT&CK负责描述攻击行为，检测平台负责产生和关联证据，SOC负责把告警转化为处置行动。

## 2. 零信任：先控制访问范围 ⭐

> 🎯 **核心理念**：不因用户位于内网就自动信任。每次访问都依据身份、设备、资源和环境风险进行明确验证与授权。

### 传统边界安全 vs 零信任
| 对比 | 传统边界安全 | 零信任 |
|------|-------------|--------|
| 信任模型 | 内网可信，外网不可信 | **从不信任** |
| 边界 | 主要依赖网络位置 | 以身份、设备和资源为控制点 |
| 认证 | 一次登录 | **持续验证** |
| 授权 | 粗粒度 | **最小权限+动态** |

### 零信任核心原则
1. **持续验证**：不只登录时验证，访问过程中持续验证身份和上下文
2. **最小权限**：只授予完成任务所需最小权限
3. **默认拒绝**：未明确允许的就拒绝
4. **微隔离**：将网络划分为细粒度区域，限制横向移动

零信任不是取消防火墙，也不是购买某一台“零信任设备”。它是一套访问控制思路，需要身份认证、设备状态检查、最小权限、微隔离、日志和持续评估共同实现。

### SDP 软件定义边界（Software-Defined Perimeter）📌
- 零信任的一种实现方式
- 隐藏目标资源，只有认证后才能访问
- BeyondCorp 是 Google 的零信任实现

### 微隔离（Microsegmentation）📌
- 将内网划分为极细粒度的安全区域
- 限制**东西向流量**（服务器间横向移动）
- 南北向流量=进出数据中心的流量；东西向流量=数据中心内部流量

## 3. 攻击框架：描述攻击者行为 ⭐

> MITRE ATT&CK 是攻击者行为知识库。**战术**表示攻击者想达到的目标，**技术**表示实现目标的方法。

### Kill Chain（攻击杀伤链）⭐
洛克希德·马丁提出的7阶段攻击模型：
```
侦察 → 武器化 → 投递 → 利用 → 安装 → 命令控制(C2) → 行动
```

| 阶段 | 说明 | 防御 |
|------|------|------|
| 侦察 | 收集目标信息 | 信息最小化 |
| 武器化 | 制作攻击载荷 | — |
| 投递 | 发送载荷(钓鱼/漏洞) | 邮件过滤/WAF |
| 利用 | 触发漏洞 | 补丁/加固 |
| 安装 | 植入后门 | EDR/白名单 |
| 命令控制(C2/C&C) | 远程控制 | 流量检测/DNS监控 |
| 行动 | 窃取/破坏 | DLP/监控 |

### ATT&CK vs Kill Chain
- Kill Chain：用线性阶段解释一次攻击如何推进，适合建立全局视角
- ATT&CK：用战术和技术矩阵描述具体行为，适合设计检测规则和评估防御覆盖

例如，“钓鱼附件”可出现在投递阶段；恶意程序创建计划任务属于持久化技术；使用合法管理员账号访问其他主机属于横向移动中的具体行为。检测团队可按 ATT&CK 技术检查是否有对应日志和规则。

## 4. 威胁情报：给检测增加外部线索 ⭐

> 威胁情报(CTI)是关于攻击者的知识，用于**提前预警和防护**。

| 类型 | 说明 |
|------|------|
| **战略情报** | 高级趋势（谁在攻击什么行业） |
| **战术情报** | 攻击者TTPs（战术/技术/流程） |
| **运营情报** | 具体攻击事件预警 |
| **技术情报** | IOC指标（IP/域名/哈希） |

### IOC（妥协指标）⭐
- 恶意IP、域名、文件哈希、注册表键等
- 用于检测是否被入侵
- 格式：STIX/TAXII（情报共享标准）

### IOC vs TTP

| 对比 | IOC | TTP |
|------|-----|-----|
| 含义 | 恶意 IP、域名、文件哈希等具体指标 | 攻击者采用的战术、技术和过程 |
| 变化速度 | 容易更换，时效较短 | 行为习惯相对稳定 |
| 用途 | 快速匹配已知威胁 | 设计行为检测和分析攻击路径 |

> 💡 文件哈希变化后，旧 IOC 可能失效；但攻击者仍可能使用相同的凭据窃取、持久化或横向移动方法。因此检测不能只依赖 IOC。

### 情报共享标准 📌
- **STIX**：情报描述语言
- **TAXII**：情报传输协议
- **MISP**：开源情报共享平台

## 5. 检测与运营：形成处置闭环

### SIEM（安全信息与事件管理）⭐
- 集中收集日志/事件 → 关联分析 → 告警
- 代表：Splunk、ELK+Elastic Security
- 核心：**关联分析**发现复合攻击

### SOC（安全运营中心）⭐
- 7×24小时安全监控响应团队
- 职能：监控→检测→分析→响应→恢复

### EDR/XDR 📌
| 类型 | 说明 |
|------|------|
| **EDR** | 端点检测与响应（终端级） |
| **XDR** | 扩展检测与响应（跨端点+网络+云） |
| **NTA/NDR** | 网络流量分析/检测 |

### SOAR 📌
- 安全编排自动化与响应
- 用剧本(playbook)自动化处理告警

### 平台与组织的关系

| 名称 | 核心职责 | 不是 |
|------|----------|------|
| **EDR** | 采集端点行为，检测和响应终端威胁 | 完整的跨系统运营中心 |
| **NDR** | 分析网络流量和通信行为 | 主机文件与进程审计工具 |
| **SIEM** | 汇聚多源日志并关联分析 | 自动完成所有事件处置 |
| **SOAR** | 按剧本编排查询、隔离、通知等动作 | 替代人工判断的万能系统 |
| **SOC** | 由人员、流程和技术组成的安全运营能力 | 某一款软件产品 |

### 4A 体系 ⭐
| A | 说明 |
|---|------|
| 账号(Account) | 统一账号管理 |
| 认证(Authentication) | 统一身份认证 |
| 授权(Authorization) | 统一权限管理 |
| 审计(Audit) | 统一安全审计 |

## 6. 贯穿案例：从钓鱼到处置

某财务员工收到伪造供应商邮件并打开附件，恶意程序窃取登录令牌，随后访问文件服务器并尝试批量读取合同。

### 预防与限制

- 零信任策略检查用户、设备合规状态和访问资源；
- MFA 降低单一口令泄露风险；
- 微隔离限制财务终端直接访问非必要服务器；
- 最小权限限制账号可读取的数据范围。

### 检测与分析

1. EDR 发现邮件程序启动异常脚本和令牌访问行为；
2. NDR 发现终端首次连接异常域名并产生横向通信；
3. 身份日志显示同一账号在异常位置登录；
4. SIEM 将端点、网络和身份日志关联为高风险告警；
5. SOC 分析员使用 ATT&CK 标注执行、凭据访问和横向移动行为；
6. 威胁情报提供恶意域名 IOC，帮助查询其他受影响终端。

### 响应与恢复

- SOAR 剧本可自动查询资产信息、封禁已确认恶意域名并通知负责人；
- 分析员确认影响后隔离终端、撤销令牌、重置凭据并阻断横向访问；
- 从日志和证据还原攻击路径，修复邮件策略和访问控制；
- 将本次攻击行为转换为新的检测规则和演练场景。

这个闭环说明：IOC 提供线索，ATT&CK描述行为，SIEM关联证据，SOAR执行标准化动作，SOC负责判断和协调处置。

## 7. 易错辨析

### 易错1：零信任是否等于永不信任

- 零信任不是拒绝所有访问，而是不依据网络位置自动授予信任
- 合法访问经过明确验证、最小授权和持续评估后仍可正常进行

### 易错2：ATT&CK vs Kill Chain

- Kill Chain 强调一次攻击的线性阶段
- ATT&CK 强调可复用的战术和技术矩阵
- 两者可以结合使用，不是互相替代关系

### 易错3：SIEM vs SOC

- SIEM 是日志汇聚和关联分析平台
- SOC 是人员、流程和技术组成的运营能力
- 购买 SIEM 不等于已经建立 SOC

### 易错4：IOC vs TTP

- IOC 是具体可匹配指标，变化快
- TTP 是攻击行为方式，更适合构建长期行为检测

## 8. 真题示例及解析

### 真题1（上午·单选）

> 某企业集中收集终端、网络和身份日志，并通过规则关联发现同一账号的异常登录与批量文件访问。承担“多源日志关联分析”的核心系统是？
> A. EDR  B. SIEM  C. VPN  D. DLP

**答案：B**

**解析**：SIEM 的核心是汇聚多源日志并进行关联分析。EDR聚焦端点行为；VPN提供安全通信隧道；DLP主要防止敏感数据泄露。

---

### 真题2（下午·简答）

> 某员工账号被盗后从异常设备登录，并访问多台内部服务器。企业已有防火墙，但没有发现该行为。请从零信任和安全运营角度提出五项改进措施。

**参考答案与评分点：**

1. 启用 MFA，并结合设备状态、登录位置和风险进行持续验证。（2分）
2. 按最小权限授权并实施微隔离，限制账号横向访问范围。（2分）
3. 部署端点和网络检测，采集进程、身份与流量行为。（2分）
4. 使用 SIEM 汇聚多源日志，关联异常登录和横向访问事件。（2分）
5. 建立 SOC 分析与响应流程，使用 SOAR 剧本辅助隔离终端、撤销令牌和通知负责人。（2分）

## 9. 考点速记
- 零信任：不因位于内网而自动信任，强调明确验证、最小权限和持续评估
- Kill Chain 7阶段：侦察→武器化→投递→利用→安装→**C2**→行动
- ATT&CK 用战术和技术描述攻击行为；Kill Chain 提供线性阶段视角
- IOC 是 IP、域名、哈希等具体指标；TTP 是相对稳定的攻击行为方式
- SIEM=日志集中+关联分析；SOC=人员+流程+技术的运营能力
- EDR终端级；XDR跨端点+网络+云
- SOAR按剧本编排响应动作，不能替代人工判断
- 4A=账号/认证/授权/审计
- 东西向流量=内部横向；南北向=进出数据中心
