# 网络安全体系与网络安全模型

> 🎯 **形象比喻**：网络安全体系像"城市防护体系"——不是单一城墙，而是城墙(边界)+巡逻(监测)+应急预案(响应)+法规(管理)的综合体系。

## 网络安全体系概述

- **概念**：由组织、管理、技术等要素构成的有机整体，实现安全保障目标
- **特征**：整体性、相关性、层次性、适应性
- **用途**：指导安全建设，避免"头痛医头"的碎片化防护

## 9大安全模型（高频考点）

### 1. BLP 机密性模型
- 保护**机密性**，规则：**下读上写**
- 防信息从高密级流向低密级（泄密）
- 详见[访问控制-安全模型](../fundamentals/models)

### 2. Biba 完整性模型
- 保护**完整性**，规则：**上读下写**（与BLP相反）
- 防低完整性数据污染高完整性数据

### 3. 信息流模型(Information Flow Model)
- 关注信息在主体/客体间的**流动方向**
- 不只看单次访问，而是追踪信息传播路径
- 用类型检查确保信息不违规流动

### 4. 信息保障模型(Information Assurance Model)
- 经典模型：**PDR**(Protection-Detection-Response) → **PDRR**(加Recovery) → **APPDRR**(加Analysis)
- 核心：防护→检测→响应→恢复的闭环
- 强调"安全是过程，不是状态"

### 5. 能力成熟度模型(CMM)
- 评估组织安全过程的成熟度，5级
- 1非正式→2计划跟踪→3充分定义→4量化控制→5持续改进
- SSE-CMM 是安全工程专用成熟度模型

### 6. 纵深防御模型(Defense in Depth)
- **多层防护**：网络边界→网络→主机→应用→数据
- 单层被突破不致全盘崩溃
- 每层独立防护，互为补充

### 7. 分层防护模型(Layered Protection)
- 类似纵深防御，强调**按OSI层次或防护层次分层**
- 物理层→数据链路层→网络层→传输层→应用层
- 每层部署对应安全措施

### 8. 等级保护模型
- 按**信息重要性分级保护**
- 中国特色：等保1.0/2.0，五级
- 详见[等保2.0](../management/level-protection)

### 9. 网络生存模型(Survivability)
- 系统在**遭受攻击/故障时仍能提供核心服务**
- 关键能力：**抵抗、识别、恢复、适应**
- 3R：Resistance(抵抗)、Recognition(识别)、Recovery(恢复)

## PDR/PDRR/APPDRR 模型对比

| 模型 | 环节 | 特点 |
|------|------|------|
| **PDR** | 防护→检测→响应 | 基础闭环 |
| **PDRR** | 防护→检测→响应→**恢复** | 加恢复 |
| **APPDRR** | **分析**→防护→策略→检测→响应→恢复 | 加前置分析 |

> 💡 演进趋势：从"被动防护"→"主动检测响应"→"自适应闭环"

## 网络安全体系建设原则

1. **同步原则**：安全与业务同步规划、建设、运行
2. **最小特权**：授予最小必要权限
3. **纵深防御**：多层独立防护
4. **默认安全**：默认配置安全
5. **失败安全**：异常时进入安全状态
6. **安全审计**：可追溯

## 网络安全体系框架组成

```
组织体系 — 安全组织架构、岗位职责
管理体系 — 制度、流程、人员培训
技术体系 — 防护/检测/响应/恢复技术
基础设施 — PKI、密钥管理、安全运营中心(SOC)
运营体系 — 日常运维、应急响应、持续改进
```

## 5. 易错辨析

### 易错1：PDR→PDRR→APPDRR演进
- PDR：防护→检测→响应
- PDRR：加**恢复**
- APPDRR：加**分析**(前置)

### 易错2：纵深防御 vs 分层防护
- 纵深防御：多层独立防护(边界/网络/主机/应用/数据)
- 分层防护：按OSI层次分层
- 类似但角度不同

### 易错3：网络生存3R
- 抵抗(Resistance)/识别(Recognition)/恢复(Recovery)

## 6. 真题示例及解析

### 真题1（上午·单选）

> PDRR 模型相比 PDR 增加了哪个环节？
> A. 分析  B. 恢复  C. 防护  D. 检测

**答案：B**

**解析**：PDR=防护→检测→响应；PDRR加**恢复**(Recovery)。APPDRR再加分析。

---

### 真题2（上午·单选）

> 纵深防御的核心思想是？
> A. 单层强防护  B. 多层防护，单层突破不致全崩  C. 只防网络层  D. 只防应用层

**答案：B**

**解析**：纵深防御多层防护，单层被突破仍有其他层保护。

## 7. 考点速记
- BLP保机密性(下读上写)，Biba保完整性(上读下写)
- 信息流模型关注**信息流动方向**
- PDR→PDRR(加恢复)→APPDRR(加分析)
- 纵深防御=多层防护，单层突破不致全崩
- 网络生存模型3R：**抵抗/识别/恢复**
- 等级保护模型按信息重要性分级
- 安全建设原则：**同步规划、最小特权、纵深防御**
