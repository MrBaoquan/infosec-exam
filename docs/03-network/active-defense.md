# 网络安全主动防御技术

> 🎯 **形象比喻**：主动防御像"主动出击"——不只被动防守（防火墙等攻击来），而是主动设陷阱(蜜罐)、预判威胁(威胁情报)、白名单只放行已知安全。

## 主动防御 vs 被动防御

| 对比 | 被动防御 | 主动防御 |
|------|----------|----------|
| 思路 | 等攻击来再处理 | 主动发现/诱骗/预判 |
| 时机 | 事后 | 事前/事中 |
| 代表 | 防火墙、IDS | 蜜罐、威胁情报、白名单 |

## 9大主动防御技术

### 1. 入侵阻断技术
- 在入侵检测基础上**实时阻断**
- IPS 是典型代表（串联部署）
- 比IDS更进一步：IDS只告警，IPS告警+阻断

### 2. 软件白名单技术
- **只允许已知安全的软件运行**，其余一律阻止
- 与黑名单相反：默认拒绝，白名单内的才放行
- 优势：可有效防未知恶意代码（0day）
- 劣势：维护成本高，新软件需及时加白

### 3. 网络流量清洗
- 对流量进行**深度检测和过滤**
- 常用于防 DDoS：清洗中心过滤恶意流量，只放行正常流量
- 部署：云端清洗中心 + 本地防护设备

### 4. 可信计算技术
- 从**硬件根**建立信任链
- TPM(Trusted Platform Module) 可信平台模块
- 信任链：CRTM→BIOS→OS→应用，逐级度量
- 核心：**度量→存储→报告**（证明平台可信）

### 5. 数字水印技术
- 在数据中嵌入**隐藏标记**，用于版权保护/溯源
- 特性：不可见性、鲁棒性（抗篡改）、安全性
- 应用：文档防泄露溯源、图片版权、视频保护

### 6. 网络攻击陷阱技术（蜜罐）
- **伪装靶机**吸引攻击，收集攻击手法
- 类型：
  - **低交互蜜罐**：模拟服务，风险低
  - **高交互蜜罐**：真实系统，信息多但风险高
- 价值：预警、收集情报、转移攻击注意力

### 7. 入侵容忍及系统生存技术
- **系统被攻击后仍能提供核心服务**
- 关键：冗余、容错、故障隔离
- 3R能力：抵抗(Resistance)、识别(Recognition)、恢复(Recovery)

### 8. 隐私保护技术
- **匿名化**：去除直接标识符
- **假名化**：用假名替换真实身份
- **差分隐私**：加入噪声，使数据无法定位个体
- **数据脱敏**：敏感字段替换/屏蔽

### 9. 网络安全前沿技术
| 技术 | 说明 |
|------|------|
| **网络威胁情报(CTI)** | 共享攻击指标(IOC)，提前预警 |
| **域名服务安全保障** | DNSSEC、DoH(DNS over HTTPS) |
| **同态加密** | 密文上直接计算，不解密 |

## 考点速记
- 主动防御：蜜罐/白名单/可信计算/流量清洗
- **白名单**只放行已知安全（防0day）；**黑名单**拦已知恶意
- 可信计算：**TPM硬件根**，信任链逐级度量
- 蜜罐：低交互(模拟)vs高交互(真实)
- 入侵容忍3R：**抵抗/识别/恢复**
- 数字水印：**不可见性+鲁棒性**
- 同态加密：**密文上直接计算**
- 差分隐私：**加噪声**保护个体
