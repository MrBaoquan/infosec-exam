# 网络攻击与防护

## 攻击分类总览

| 类别 | 目的 | 代表 |
|------|------|------|
| **被动攻击** | 窃听（不修改） | 网络嗅探、流量分析 |
| **主动攻击** | 篡改/伪造/中断 | 篡改、伪造、DoS、重放 |

> 被动攻击难发现但可防（加密）；主动攻击可发现但难完全防。

## DoS/DDoS 拒绝服务

- **DoS**：单点发起；**DDoS**：分布式（僵尸网络）发起
- 目的：耗尽带宽/资源/连接，使服务不可用

### 常见类型

| 类型 | 原理 |
|------|------|
| **SYN Flood** | 大量半连接耗尽 TCP 连接表 |
| **Smurf** | ICMP 广播放大 |
| **Land** | 源地址=目的地址的自攻击 |
| **放大攻击** | 利用 DNS/NTP 等无认证服务放大流量 |
| **慢速攻击** | Slowloris 等慢速占用连接 |

### 防护
- 流量清洗、CDN 分流、黑洞路由
- SYN Cookie（防 SYN Flood）
- 连接限速、异常流量检测

## MITM 中间人攻击

- 攻击者冒充双方，转发/篡改通信
- 常见：ARP 欺骗、DNS 欺骗、SSL 剥离
- 防护：**证书验证**、**HSTS**、数字签名、VPN

## 重放攻击 Replay

- 截获并重发合法报文（如登录请求）
- 防护：**时间戳**、**序列号/Nonce**、**一次性令牌**

## 社会工程学

- 利用**人性弱点**（而非技术漏洞）
- 钓鱼邮件、伪装身份、尾随
- 防护：**安全意识培训**（技术手段无法完全防御）

## APT 高级持续性威胁

- 定向、长期、隐蔽的攻击（国家级/企业级）
- 特征：长期潜伏、多阶段、定制化
- 典型：鱼叉钓鱼 → 渗透 → 横向移动 → 数据窃取

## 渗透测试流程（高频）

```
1. 信息收集 → 2. 扫描探测 → 3. 漏洞利用 → 4. 后渗透 → 5. 报告
```

- 与黑客攻击区别：**经授权**、**不破坏**、**出报告**
- 分类：**黑盒**（不知内部）/ **白盒**（完全知情）/ **灰盒**（部分知情）

## 5. 易错辨析

### 易错1：被动 vs 主动攻击
- **被动**：只窃听不修改（嗅探/流量分析），**防：加密**
- **主动**：篡改/伪造/中断（篡改/DoS/重放），**防：认证+完整性**
- 考题："哪个是被动攻击"→嗅探/窃听

### 易错2：SYN Flood 防御
- SYN Flood 利用TCP三次握手，发大量SYN不回ACK，耗尽半连接队列
- 防：**SYN Cookie**（不分配半连接资源，编码到序列号）

### 易错3：重放攻击防御
- 重放=截获合法报文重发（如重放登录请求）
- 防：**时间戳/Nonce/一次性令牌**
- 加密/签名**不能**防重放（重放的是合法加密报文）

### 易错4：渗透测试 vs 黑客攻击
- 区别：**经授权、不破坏、出报告**
- 分类：黑盒(不知)/白盒(全知)/灰盒(部分)

## 6. 真题示例及解析

### 真题1（上午·单选）

> 下列属于被动攻击的是？
> A. 篡改  B. 伪造  C. 窃听  D. DoS

**答案：C**

**解析**：窃听是被动攻击（只监听不修改）。篡改/伪造/DoS都是主动攻击。被动攻击难发现但可防（加密），主动攻击可发现但需多手段防。

---

### 真题2（上午·单选）

> SYN Flood 攻击利用了TCP三次握手的什么机制？
> A. 全连接耗尽  B. 半连接耗尽  C. FIN包伪造  D. 序列号预测

**答案：B**

**解析**：SYN Flood发大量SYN不完成第三次握手，耗尽服务器**半连接队列**。防：SYN Cookie。

---

### 真题3（上午·单选）

> 防御重放攻击的有效措施是？
> A. 加密  B. 时间戳/Nonce  C. 数字签名  D. 哈希

**答案：B**

**解析**：重放防时间戳/序列号/一次性令牌。加密/签名不能防重放（重放的是合法加密签名报文）。

---

### 真题4（上午·单选）

> 渗透测试与黑客攻击的根本区别是？
> A. 使用工具不同  B. 经授权、不破坏、出报告  C. 技术水平不同  D. 目标不同

**答案：B**

**解析**：渗透测试：经授权、不破坏系统、产出报告。分黑盒/白盒/灰盒。核心是"合法合规的安全评估"。

## 7. 考点速记

- 被动攻击=窃听（防：加密）；主动攻击=篡改/DoS/重放
- SYN Flood 防 SYN Cookie；Smurf 是 ICMP 放大
- 中间人攻击防：证书验证+HSTS
- 重放攻击防：**时间戳/Nonce/一次性令牌**
- 社会工程防：**安全意识**（非技术）
- 渗透测试：黑盒/白盒/灰盒；经授权、不破坏、出报告
