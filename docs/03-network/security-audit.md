# 网络安全审计技术

> 🎯 **形象比喻**：安全审计像"银行的监控录像+交易记录"——事后可追溯谁在什么时候做了什么。

## 审计概念

对网络信息系统的**安全相关活动**进行记录、分析和审查，用于合规检查、事件追溯、取证。

> 《网络安全法》要求日志留存**≥6个月**。

## 审计系统类型

| 类型 | 审计对象 | 典型 |
|------|----------|------|
| **网络通信审计** | 网络流量 | 抓包分析 |
| **操作系统审计** | 系统日志/事件 | Windows事件日志、Linux syslog |
| **数据库审计** | SQL操作 | 数据库审计系统 |
| **应用系统审计** | 应用行为 | Web访问日志 |
| **运维安全审计** | 运维操作 | 堡垒机 |

## 审计数据采集技术

### 网络流量采集
| 方法 | 说明 |
|------|------|
| **交换机端口镜像(SPAN)** | 将端口流量复制到审计口 |
| **网络嗅探** | 网卡混杂模式抓包 |
| **Tcpdump** | 命令行抓包工具 |
| **Wireshark** | 图形化抓包分析工具 |

### 系统日志采集
| 协议/工具 | 说明 |
|----------|------|
| **Syslog** | 标准日志协议(UDP 514) |
| **SNMP** | 网络设备状态/告警 |
| **FTP** | 日志文件传输 |

## 审计数据分析技术

| 方法 | 说明 |
|------|------|
| **字符串匹配** | 关键字检索（如"failed login"） |
| **全文搜索** | 大量日志快速检索 |
| **数据关联** | 多源日志关联分析 |
| **统计报表** | 趋势分析、异常统计 |
| **可视化分析** | 图表展示，发现异常模式 |

## 审计数据保护

- **分权管理**：审计员与系统管理员分离
- **强制访问**：审计数据只读，防篡改
- **加密**：审计数据加密存储
- **完整性保护**：哈希校验防篡改
- **备份**：审计数据定期备份

> 🔑 **审计数据防篡改是关键**——攻击者首先会试图删除/篡改日志。

## Tcpdump 常用命令

```bash
# 抓指定端口流量
tcpdump -i eth0 port 80
# 抓指定IP
tcpdump -i eth0 host 192.168.1.1
# 保存到文件
tcpdump -w capture.pcap
# 读取抓包文件
tcpdump -r capture.pcap
```

## 考点速记
- 审计系统5类：网络/操作系统/数据库/应用/运维
- 网络流量采集：**端口镜像(SPAN)** + 嗅探
- 日志采集：**Syslog(UDP 514)** + SNMP
- 审计工具：**Tcpdump(命令行)** / **Wireshark(图形)**
- 审计数据保护：**分权管理、防篡改、加密**
- 日志留存 **≥6个月**（网安法）
- 审计员与管理员**职责分离**
