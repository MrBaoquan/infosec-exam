# 无线网络安全

## 1. 为什么无线网络更难保护

有线网络需要接触网线或交换机端口，无线信号却会穿过办公室墙壁。攻击者即使不进入公司，也可能在停车场监听信号、伪造热点或尝试破解弱口令。

> 🎯 **形象比喻**：有线网络像在管道里送信，无线网络像用广播喇叭传递。任何处在信号范围内的人都能听见，因此必须依靠可靠的认证和加密区分合法用户。

无线安全要同时解决三个问题：

1. **接入认证**：确认设备或用户是否有权加入网络；
2. **链路加密**：防止无线帧被窃听或篡改；
3. **热点可信**：避免用户连接攻击者伪造的接入点。

## 2. 802.11 基本角色

| 术语 | 全称 | 作用 |
|------|------|------|
| **STA** | Station | 无线终端，如手机和笔记本 |
| **AP** | Access Point | 无线接入点，连接终端与有线网络 |
| **SSID** | Service Set Identifier | 无线网络名称，不是安全凭据 |
| **BSSID** | Basic Service Set Identifier | 通常对应 AP 无线接口的 MAC 地址 |

终端通常先发现 SSID，再与 AP 完成认证、关联和密钥协商，之后才能传输受保护的数据帧。

## 3. 安全协议演进

| 协议 | 核心机制 | 安全评价 |
|------|----------|----------|
| **WEP** | RC4 + 短 IV | IV 空间小且密钥管理薄弱，已淘汰 |
| **WPA** | TKIP，兼容旧硬件 | 过渡方案，已不推荐 |
| **WPA2** | AES-CCMP | 长期主流；安全性取决于模式和口令 |
| **WPA3** | SAE + 更强的管理帧保护要求 | 抵抗离线字典攻击能力更强，优先选用 |
| **WAPI** | 采用我国无线局域网安全机制 | 国密相关场景可能考查 |

> ⭐ 高频结论：WEP 已不安全；WPA 是过渡方案；WPA2 使用 AES-CCMP；WPA3-Personal 使用 SAE 改进口令认证。

### Personal 与 Enterprise

| 模式 | 认证方式 | 适用场景 | 主要风险 |
|------|----------|----------|----------|
| **Personal** | 所有终端共享 PSK 口令 | 家庭、小型办公室 | 一人泄露口令影响全网，难以单独撤销 |
| **Enterprise** | 802.1X + EAP + RADIUS，按用户或设备认证 | 企业、学校 | 配置复杂，需要保护认证服务器和证书 |

企业模式的简化流程：

```text
终端申请接入 → AP 转发认证消息 → RADIUS 验证身份 → 为合法会话生成密钥 → 放行网络访问
```

AP 在此过程中承担认证者角色，RADIUS 是后端认证服务器。不要把 802.1X 误认为某种加密算法，它是端口接入控制框架。

## 4. 四次握手解决什么问题

WPA2 中，终端和 AP 已拥有由认证过程得到的主密钥材料，四次握手用于：

- 确认双方都掌握主密钥材料；
- 结合随机数和双方地址派生会话密钥；
- 安装后续数据加密与完整性保护所需的密钥。

> 💡 四次握手不会在空中直接发送 PSK 明文。弱口令仍可能被攻击者通过捕获握手数据后进行字典猜测，因此长且随机的口令仍然重要。

## 5. 典型无线攻击

### 邪恶双胞胎 Evil Twin

攻击者架设与企业同名的伪造热点，诱导用户连接，再实施钓鱼或中间人攻击。只核对 SSID 无法识别真假 AP，因为 SSID 可以任意伪造。

### 伪造 AP Rogue AP

员工私接无线路由器或攻击者接入非法 AP，使企业边界策略被绕过。Rogue AP 强调“未经授权接入企业网络”，Evil Twin 强调“冒充可信热点诱骗终端”。

### 解除认证攻击

攻击者伪造管理帧让终端掉线，可用于拒绝服务或迫使终端重新连接。管理帧保护 PMF 可降低此类伪造攻击风险。

### 弱口令猜测

WPA2-Personal 使用弱 PSK 时，攻击者可捕获握手数据后离线尝试候选口令。防护重点是高强度随机口令、WPA3-SAE 或企业级认证。

## 6. 企业无线防护方案

某公司需要为员工和访客提供无线网络，可按以下方式设计：

1. 员工网采用 WPA3-Enterprise，暂不支持时使用 WPA2-Enterprise；
2. 使用 802.1X、EAP 和 RADIUS，为每名员工独立认证；
3. 员工、访客和物联网设备使用不同 SSID，并映射到不同 VLAN；
4. 访客只能访问互联网，不能进入内部业务网；
5. 验证认证服务器证书，防止终端把凭据交给伪造热点；
6. 启用无线入侵检测，发现 Rogue AP、Evil Twin 和异常断线；
7. 禁用 WEP、WPA-TKIP 和 WPS，启用管理帧保护；
8. 定期更新 AP 固件，并更换离职人员可能知晓的共享口令。

## 7. 易错辨析

### 易错1：隐藏 SSID 是否等于安全

- 隐藏 SSID 不能提供可靠机密性，网络名称仍可能从无线帧中推断
- 真正的安全依赖认证、加密和访问控制

### 易错2：WPA2 是否一定安全

- WPA2-AES 的机制可靠，但弱 PSK、错误证书验证和不安全配置仍会造成风险
- 企业环境优先使用按用户认证，而不是全员共享口令

### 易错3：802.1X vs 802.11

- 802.11 定义无线局域网通信
- 802.1X 提供基于端口的网络接入控制
- RADIUS 常作为 802.1X 的后端认证服务器

### 易错4：Rogue AP vs Evil Twin

- Rogue AP：未经授权连接到企业网络
- Evil Twin：冒充可信热点诱骗用户连接

## 8. 真题示例及解析

### 真题1（上午·单选）

> 某企业希望无线网络按员工账号进行独立认证，员工离职后可单独撤销其权限。最合适的方案是？
> A. 隐藏SSID  B. WPA2-Personal共享口令  C. 802.1X结合RADIUS  D. MAC地址白名单

**答案：C**

**解析**：802.1X结合RADIUS可按用户或设备独立认证和撤销。共享PSK无法单独撤销某名员工；隐藏SSID不提供可靠认证；MAC地址可伪造，不能作为主要身份凭据。

---

### 真题2（下午·简答）

> 某公司员工与访客共用一个采用短共享口令的无线网络，近期发现办公区附近存在同名热点。请指出两项风险并提出四项整改措施。

**参考答案与评分点：**

1. 短共享口令可能被猜测，且人员变化后难以单独撤销。（2分）
2. 同名热点可能是 Evil Twin，导致凭据泄露或中间人攻击。（2分）
3. 员工网使用 WPA3-Enterprise 或 WPA2-Enterprise，结合 802.1X 和 RADIUS 独立认证。（2分）
4. 员工网与访客网使用不同 SSID 和 VLAN，访客网禁止访问内部业务区。（2分）
5. 要求终端验证认证服务器证书，并部署无线入侵检测发现伪造或非法 AP。（2分）

## 9. 考点速记

- WEP 已淘汰；WPA 是过渡方案；WPA2 使用 AES-CCMP；WPA3-Personal 使用 SAE
- Personal 使用共享 PSK；Enterprise 使用 802.1X、EAP 和 RADIUS 独立认证
- SSID 可伪造，隐藏 SSID 不能替代认证和加密
- Evil Twin 冒充热点；Rogue AP 是未经授权接入企业网络的 AP
- 企业无线网应按员工、访客和物联网设备分区，并验证认证服务器证书
