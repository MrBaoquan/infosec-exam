# 安全管理与运维

## 信息安全管理体系 ISMS

- 基于 **ISO/IEC 27001** 标准
- 核心方法：**PDCA 循环**（Plan-Do-Check-Act）
  - Plan：建立 ISMS（风险评估、策略制定）
  - Do：实施运行 ISMS
  - Check：监视评审 ISMS
  - Act：改进 ISMS

| 标准 | 内容 |
|------|------|
| **ISO 27001** | ISMS 要求（可认证） |
| **ISO 27002** | 安全控制措施指南（实践） |

## 安全管理制度体系

```
1. 安全策略（总纲）
2. 管理规定（各领域细则）
3. 操作规程（具体操作步骤）
4. 记录表单（执行记录）
```

## 人员安全管理

| 环节 | 措施 |
|------|------|
| 入职 | 背景调查、保密协议、安全培训 |
| 在岗 | 定期培训、权限审查、行为审计 |
| 离职 | 立即收回权限、交接、保密义务延续 |

> **职责分离原则**：关键操作需多人完成（如开发与运维分离，防单点作恶）。

## 安全运维管理

| 领域 | 要点 |
|------|------|
| **变更管理** | 变更申请→评估→审批→测试→实施→回滚 |
| **配置管理** | 配置基线、版本控制 |
| **补丁管理** | 评估→测试→部署→验证 |
| **漏洞管理** | 监控→评估→修复 |
| **日志管理** | 集中收集、定期分析、留存(≥6个月) |

### 日志留存要求
- 《网络安全法》规定：网络日志**留存不少于 6 个月**

## 物理安全

| 维度 | 措施 |
|------|------|
| 机房选址 | 防灾、防潮、供电 |
| 物理访问 | 门禁、登记、监控 |
| 防火防水 | 灭火系统、漏水检测 |
| 供电 | UPS、双路市电、发电机 |
| 防静电 | 防静电地板、接地 |

## 供应链安全

- 供应商评估与准入
- 软件供应链：第三方组件漏洞（如 Log4j）
- SBOM 软件物料清单

## 5. 易错辨析

### 易错1：ISMS 标准
- **ISO 27001**：ISMS**要求**（可认证）
- **ISO 27002**：安全**控制措施**指南
- 核心方法：**PDCA**循环

### 易错2：日志留存
- 《网络安全法》：日志**≥6个月**
- 不是3个月或1年

### 易错3：职责分离
- 开发与运维分离
- 审计员与管理员分离
- 防**单点作恶**

### 易错4：变更管理
- 流程：申请→评估→审批→**测试**→实施→**回滚**
- 必须先测试，必须有回滚方案

## 6. 真题示例及解析

### 真题1（上午·单选）

> ISMS 基于 ISO 27001，其核心方法是？
> A. 瀑布模型  B. PDCA循环  C. 敏捷开发  D. 六西格玛

**答案：B**

**解析**：ISMS基于ISO/IEC 27001，核心是PDCA(Plan-Do-Check-Act)循环。27002是控制指南。

---

### 真题2（上午·单选）

> 《网络安全法》要求网络日志留存不少于？
> A. 3个月  B. 6个月  C. 1年  D. 2年

**答案：B**

**解析**：《网络安全法》第21条规定日志留存不少于6个月。

---

### 真题3（上午·单选）

> 变更管理流程中，部署前必须？
> A. 直接全量部署  B. 在测试环境验证  C. 忽略  D. 只通知用户

**答案：B**

**解析**：变更管理：申请→评估→审批→**测试**→实施→回滚。必须先测试验证。

## 7. 考点速记

- ISMS 基于 **ISO 27001**，核心是 **PDCA 循环**
- 27001=要求（可认证），27002=控制指南
- 人员安全：入职背调、离职收权、职责分离
- **日志留存≥6个月**（网安法要求）
- 变更管理：申请→评估→审批→测试→实施→**回滚**
- 职责分离：开发与运维分离
