# 六、信息安全管理

## 风险评估

风险 = 资产 × 威胁 × 脆弱性

- **资产**：有价值的信息/系统
- **威胁**：可能导致破坏的源头
- **脆弱性(漏洞)**：可被利用的弱点
- 风险值 = 影响 × 可能性

评估方法：定性(矩阵)/定量(ALE 年度预期损失)

## 等级保护 2.0（高频）

五个级别：
1. 自主保护（一般系统）
2. 指导保护
3. **监督保护**（重要系统，需定期测评）
4. 强制保护（关键系统）
5. 专控保护（特殊系统）

> 三级及以上必须**定期测评**；定级 → 备案 → 建设整改 → 测评 → 运维。

## 应急响应

六阶段：准备 → 检测 → 抑制 → 根除 → 恢复 → 总结

## BCP / DRP

| 项 | 含义 |
|----|------|
| **BCP** 业务连续性计划 | 保证关键业务不中断 |
| **DRP** 灾难恢复计划 | 灾难后恢复 IT 系统 |
| **RTO** 恢复时间目标 | 系统恢复的最长允许时间 |
| **RPO** 恢复点目标 | 可容忍的数据丢失量(时间) |
| **MTD** 最大容忍中断时间 | 业务可承受最长中断 |

> RTO ≤ MTD；RPO 决定备份频率。

## 备份策略

- 全量、增量、差异
- 备份介质异地存放；3-2-1 原则（3 份/2 介质/1 异地）

## 安全运维

- 变更管理、配置管理、补丁管理、漏洞管理
- 最小权限、职责分离

## 考点速记

- 风险 = 资产 × 威胁 × 脆弱性
- 等保 2.0 五级，三级及以上需测评；流程：定级→备案→整改→测评→运维
- 应急响应六阶段：准备/检测/抑制/根除/恢复/总结
- RTO=恢复时间，RPO=数据丢失量；RTO≤MTD
- 备份 3-2-1 原则
