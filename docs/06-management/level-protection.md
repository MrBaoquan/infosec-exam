# 网络安全等级保护 2.0

> 🎯 **形象比喻**：等保像"建筑安全等级"——不同重要性的建筑有不同抗震要求。普通住宅(1-2级)要求低，医院/指挥中心(3-4级)要求高，必须验收。

> 等保 2.0 = 依据《网络安全法》的强制安全制度，标准号 GB/T 22239-2019。

## 五个级别

| 级别 | 名称 | 保护要求 | 测评 |
|------|------|----------|:----:|
| 1 | 自主保护 | 一般系统 | 否 |
| 2 | 指导保护 | 一般系统 | 否 |
| **3** | **监督保护** | **重要系统** | **是** |
| 4 | 强制保护 | 关键系统 | 是 |
| 5 | 专控保护 | 特殊系统 | 是 |

> 🔑 **三级及以上必须定期测评**（每年一次）。

## 等保 2.0 vs 1.0 变化

| 维度 | 1.0 | 2.0 |
|------|-----|-----|
| 对象 | 信息系统 | **网络基础设施/云计算/大数据/物联网/工控** |
| 标准 | GB/T 22239-2008 | GB/T 22239-2019 |
| 理念 | 合规导向 | **主动防御+综合防护** |

## 等保工作流程（必背）

```
定级 → 备案 → 建设整改 → 测评 → 运维
```

1. **定级**：确定保护等级（专家评审）
2. **备案**：二级以上到公安机关备案（10个工作日内）
3. **建设整改**：按等级要求建设安全设施
4. **测评**：第三方测评机构测评（三级每年一次）
5. **运维**：持续运行维护、定期复查

## 等保 2.0 安全要求框架（三重防护）

```
安全计算环境 ← 安全区域边界 ← 安全通信网络
                ↑
          安全管理中心
```

| 维度 | 内容 |
|------|------|
| **安全通信网络** | 网络架构、通信传输、可信接入 |
| **安全区域边界** | 边界防护、访问控制、入侵防范 |
| **安全计算环境** | 身份鉴别、访问控制、安全审计、入侵防范、恶意代码防范 |
| **安全管理中心** | 系统管理、安全管理、审计管理 |

## 等保 2.0 通用要求 vs 云扩展

- 通用要求：传统信息系统
- 云计算扩展：责任共担、虚拟化安全、数据迁移
- 移动互联扩展：移动终端管控
- 物联网扩展：感知设备安全
- 工业控制扩展：SCADA 安全

## 5. 易错辨析

### 易错1：等保级别与测评
- 三级及以上需**定期测评**（每年）
- 备案：**二级以上**到公安机关，10个工作日内
- 五级：1自主/2指导/3监督/4强制/5专控

### 易错2：等保流程
- 定级→备案→建设整改→测评→运维
- 顺序不能乱

### 易错3：三重防护
- 安全通信网络/安全区域边界/安全计算环境 + 安全管理中心

## 6. 真题示例及解析

### 真题1（上午·单选）

> 等保2.0中需定期测评的级别是？
> A. 一级及以上  B. 二级及以上  C. 三级及以上  D. 五级

**答案：C**

**解析**：三级及以上需定期测评（每年一次）。三级=监督保护，重要系统。

---

### 真题2（上午·单选）

> 等保工作流程的正确顺序是？
> A. 定级→备案→建设整改→测评→运维  B. 备案→定级→测评→整改→运维  C. 测评→定级→备案→整改→运维  D. 定级→测评→备案→整改→运维

**答案：A**

**解析**：定级→备案→建设整改→测评→运维。先定级再备案。

---

### 真题3（上午·单选）

> 等保2.0备案的时限是？
> A. 5个工作日  B. 10个工作日  C. 30个工作日  D. 60个工作日

**答案：B**

**解析**：二级以上系统定级后10个工作日内到公安机关备案。

## 7. 考点速记

- 等保 2.0 五级，**三级及以上需定期测评**
- 流程：**定级→备案→建设整改→测评→运维**
- 备案：**二级以上**到公安机关，10 个工作日内
- 三级测评**每年一次**
- 三重防护：**通信网络/区域边界/计算环境** + 安全管理中心
- 2.0 扩展覆盖云/移动/物联网/工控
