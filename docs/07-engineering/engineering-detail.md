# 信息安全工程

> 🎯 **形象比喻**：安全工程像"盖安全大楼"——不能盖完再装防盗门（事后补），要从设计图阶段就考虑安全（**安全左移**）。

## 系统安全工程

将安全活动贯穿系统**全生命周期**：

```
需求 → 设计 → 实现 → 测试 → 部署 → 运维 → 退役
```

> 安全应**左移**（在设计阶段就考虑，而非事后补）。

## SSE-CMM 安全工程能力成熟度模型

描述组织安全工程过程能力，**5 个等级**：

| 级别 | 名称 | 特征 |
|------|------|------|
| 1 | **非正式** | 依赖个人英雄主义，不可重复 |
| 2 | **计划与跟踪** | 可重复，有基本计划 |
| 3 | **充分定义** | 标准化流程，组织级 |
| 4 | **量化控制** | 可度量，数据驱动 |
| 5 | **持续改进** | 优化改进，自适应 |

> 与 CMM 软件能力成熟度模型对应，SSE-CMM 专注安全工程。

## CC 通用准则（ISO/IEC 15408）

- 国际信息安全产品评估标准
- 三部分：
  - 第1部分：简介和一般模型
  - 第2部分：**安全功能要求**（SFR）
  - 第3部分：**安全保证要求**（SAR）

### EAL 评估保证级

| 级别 | 保证程度 |
|------|----------|
| EAL1 | 功能测试（最低） |
| EAL2 | 结构测试 |
| EAL3 | 方法测试与检查 |
| EAL4 | 系统设计测试与复查（**最高商业实用**） |
| EAL5 | 半形式化设计与测试 |
| EAL6 | 半形式化验证设计与测试 |
| EAL7 | 形式化验证（最高） |

> EAL4 是商业产品通常达到的最高级别；EAL5-7 多用于军方/高安全产品。

## 安全设计原则（Saltzer-Schroeder）

| 原则 | 含义 |
|------|------|
| **经济机制** | 简单可理解 |
| **失败安全** | 失败时进入安全状态（默认拒绝） |
| **默认安全** | 默认配置安全 |
| **纵深防御** | 多层防护 |
| **最小权限** | 仅授予必要权限 |
| **职责分离** | 关键操作多人 |
| **完全调解** | 每次访问都检查 |
| **开放设计** | 安全不依赖保密（Kerckhoffs 原则） |
| **心理可接受** | 安全措施易用 |
| **最薄弱环节** | 重视最弱点 |

## 可信计算

| 概念 | 说明 |
|------|------|
| **TPM** | 可信平台模块（硬件根可信） |
| **TCM** | 国产可信密码模块 |
| **信任链** | CRTM→BIOS→OS→应用，逐级度量 |
| **远程证明** | 向第三方证明平台可信状态 |

> 信任根：CRTM（核心度量根），硬件固化不可篡改。

## 5. 易错辨析

### 易错1：SSE-CMM 5级
- 1非正式→2计划跟踪→3充分定义→4量化控制→5持续改进

### 易错2：CC/EAL
- EAL1~EAL7
- **EAL4**：商业产品最高实用级
- EAL5-7：军方/高安全

### 易错3：Kerckhoffs原则
- 安全不依赖算法保密（开放设计）
- 安全只依赖密钥

### 易错4：安全设计原则
- 默认安全/纵深防御/失败安全/最小权限/职责分离/开放设计

## 6. 真题示例及解析

### 真题1（上午·单选）

> SSE-CMM 共有几个等级？
> A. 3级  B. 4级  C. 5级  D. 7级

**答案：C**

**解析**：SSE-CMM共5级：1非正式→2计划跟踪→3充分定义→4量化控制→5持续改进。

---

### 真题2（上午·单选）

> CC 通用准则中，商业产品通常达到的最高级别是？
> A. EAL1  B. EAL3  C. EAL4  D. EAL7

**答案：C**

**解析**：EAL4是商业产品通常达到的最高级别。EAL5-7多用于军方。

---

### 真题3（上午·单选）

> Kerckhoffs 原则的核心思想是？
> A. 算法应保密  B. 安全不应依赖算法保密  C. 密钥应公开  D. 加密越复杂越安全

**答案：B**

**解析**：安全应基于密钥保密而非算法保密。假设攻击者知道算法，安全只依赖密钥。

---

### 真题4（上午·单选）

> "系统异常时进入安全状态，默认拒绝访问"属于？
> A. 最小权限  B. 纵深防御  C. 失败安全  D. 默认安全

**答案：C**

**解析**：失败安全(Fail Safe)：异常时进入安全状态，默认拒绝。

## 7. 考点速记

- SSE-CMM **5级**：1非正式→2计划跟踪→3充分定义→4量化控制→5持续改进
- CC 三部分：简介/功能要求/保证要求；**EAL1~EAL7**
- **EAL4** 商业最高实用级；EAL5-7 军方级
- 安全原则：**默认安全、纵深防御、失败安全、最小权限、职责分离、开放设计**
- Kerckhoffs：**安全不依赖算法保密**（应假设攻击者知道算法）
- TPM 硬件根可信；信任链逐级度量
