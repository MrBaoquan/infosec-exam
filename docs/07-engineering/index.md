# 七、信息安全工程

## 系统安全工程

将安全活动贯穿系统**全生命周期**：
发现需求 → 定义系统 → 设计 → 实现 → 测试 → 部署运维 → 退役

## SSE-CMM 安全工程能力成熟度模型

描述组织安全工程过程能力，**5 个等级**：

| 级别 | 名称 | 特征 |
|------|------|------|
| 1 | 非正式 | 依赖个人 |
| 2 | 计划与跟踪 | 可重复 |
| 3 | 充分定义 | 标准化流程 |
| 4 | 量化控制 | 可度量 |
| 5 | 持续改进 | 优化改进 |

## 安全工程原则

- 默认安全(Secure by Default)
- 纵深防御(Defense in Depth)
- 失败安全(Fail Safe)
- 经济机制(Economy of Mechanism)
- 最小权限
- 职责分离

## 通用准则 CC

- ISO/IEC 15408
- 三部分：简介/安全功能要求/安全保证要求
- 评估保证级 EAL1~EAL7

## 考点速记

- SSE-CMM **5 级**：1非正式→2计划跟踪→3充分定义→4量化控制→5持续改进
- 安全工程贯穿系统全生命周期
- CC 通用准则 EAL1~EAL7
- 核心原则：默认安全、纵深防御、失败安全、最小权限
